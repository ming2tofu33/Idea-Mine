# Landing / Mine 리디자인 결정 문서

> IDEA MINE v2의 랜딩 페이지와 광맥(Mine) 페이지 리디자인 기준 문서. 이 문서는 구현 전에 페이지 구조, 장면 설계, 공통 UI 문법을 결정 완료 상태로 고정하기 위한 문서다. 상위 비주얼 원칙과 컬러/CTA 토큰은 `Premium-Space-UI-Style-Guide.md`를 기준으로 한다.

## 1. Purpose

이 문서의 목적은 이번 라운드에서 실제로 손볼 두 페이지, 즉 랜딩 페이지와 광맥(Mine) 페이지의 구조와 장면 설계를 구체적으로 고정하는 것이다. 기존 스타일 가이드가 컬러, 재질감, 모션, CTA 원칙 같은 상위 비주얼 시스템을 정의했다면, 이 문서는 그 원칙을 페이지 단위의 와이어프레임과 컴포넌트 규칙으로 번역한다.

이번 문서는 구현 계획서가 아니라 디자인 결정 문서다. 즉, 구현자가 이 문서를 읽고 추가적인 방향 결정을 하지 않아도 페이지 구조와 역할, 강조 포인트, CTA 위계, 상태 변화의 의도를 바로 이해할 수 있어야 한다.

## 2. Redesign Goals

이번 리디자인의 핵심 목표는 세 가지다.

첫째, 랜딩과 광맥 페이지를 더 이상 평범한 SaaS 레이아웃이나 카드 그리드처럼 보이지 않게 만든다. 두 페이지 모두 `Cinematic Observatory`라는 동일한 세계 안에 있어야 하며, 각각 분명한 대표 장면을 가져야 한다.

둘째, 랜딩은 단순한 소개 페이지가 아니라 우주 관측소의 입구처럼 느껴져야 한다. 세계관과 브랜드 첫인상을 강하게 전달하면서도, 실제 제품 구조와 핵심 가치가 빠르게 이해되어야 한다.

셋째, 광맥 페이지는 카드 3개를 나열한 선택 화면이 아니라, 관측 시스템이 세 개의 광맥 신호를 탐지하고 사용자가 그중 하나를 지정해 탐사로 들어가는 장면이 되어야 한다.

## 3. Shared Page Language

랜딩과 광맥 페이지는 역할이 다르지만, 같은 제품의 같은 세계 안에 있어야 한다. 두 페이지를 묶는 공통 문법은 아래와 같다.

### 3.1 Full-Bleed Atmosphere + Framed Content

배경은 화면 전체를 채우는 우주적 대기층이어야 한다. 성운, 입자, 빛 흐름, 깊이감은 페이지 전체에 살아 있어야 한다. 다만 실제 콘텐츠는 그 위에 통제된 프레임 안에 배치되어야 하며, UI는 우주 공간에 흩뿌려진 텍스트가 아니라 정밀 장비의 인터페이스처럼 보여야 한다.

### 3.2 Thin Status Rail / Command Rail

모든 주요 페이지 상단에는 두꺼운 헤더 대신 얇고 정밀한 상태줄이 들어간다. 이 상태줄은 단순 내비게이션이 아니라 시스템이 현재 작동 중임을 보여주는 HUD다. 랜딩에서는 브랜드와 진입을, 광맥 페이지에서는 현재 스캔 상태와 자원을 보여주는 역할을 한다.

### 3.3 One Dominant Stage

각 페이지에는 반드시 하나의 강한 대표 장면이 있어야 한다. 랜딩에서는 `Split Hero`, 광맥 페이지에서는 `Sector Scan Field`가 그 역할을 맡는다. 여러 박스가 같은 목소리로 배치되면 기억에 남는 장면이 사라지므로, 한 페이지에는 하나의 무대와 그것을 보조하는 정보 섬 구조를 유지한다.

### 3.4 Signal-Driven Interaction

hover, focus, selected, active 상태는 모두 `signal response`처럼 반응해야 한다. 즉, 색이 단순히 바뀌는 것이 아니라 신호선, 링, 글로우, 라벨이 살아나는 방식으로 상태가 표현되어야 한다. 이 규칙이 랜딩과 광맥 페이지를 같은 시스템처럼 묶는다.

### 3.5 Calm Information Islands

읽어야 하는 정보는 언제나 `정보 섬`처럼 묶여 있어야 한다. 배경 위에 텍스트를 흩뿌리지 않고, 패널, 카드, 모듈 안에서 정보가 정리되어 보여야 한다. 장면은 시네마틱하게 클 수 있지만, 정보는 항상 차분하고 명확하게 읽혀야 한다.

### 3.6 Contrast by Purpose

장면은 강하지만, 읽고 판단해야 하는 순간에는 콘텐츠가 앞으로 나와야 한다. 분위기와 사용성의 우선순위가 충돌할 때는 항상 콘텐츠가 우선한다. 랜딩의 메시지, 광맥 페이지의 상세 패널, CTA의 위계는 모두 이 규칙을 따른다.

## 4. Landing Page Wireframe

랜딩 페이지의 역할은 `세계관 + 제품 이해 + 진입 유도`를 동시에 수행하는 하이브리드 입구다. 단순 마케팅 랜딩이 아니라, 사용자가 관측소 안으로 들어오는 첫 장면처럼 느껴져야 한다.

### 4.1 Status Rail / Top Navigation

페이지 최상단에는 얇은 상태줄을 둔다.

- 좌측: `IDEA MINE` 워드마크와 짧은 시스템 라벨
- 중앙 또는 보조 영역: 제품 성격을 암시하는 짧은 시스템 텍스트
- 우측: `Sign in`, `Enter Mine`

이 영역은 헤더라기보다 관측 장비가 켜져 있는 패널처럼 느껴져야 한다. 시안/스틸 계열의 반응을 기본으로 하며, 핑크는 최소한으로만 사용한다.

### 4.2 Split Hero

랜딩의 대표 장면이다. 좌우 분할 구조를 사용하되, 좌측은 카피와 행동, 우측은 실제 제품 프리뷰를 담당한다.

좌측에는 아래 정보가 들어간다.

- 작은 kicker 또는 시스템 라벨
- 강한 headline
- 짧은 설명 문장
- 1차 CTA와 2차 CTA
- 아주 짧은 proof line

우측에는 정적인 스크린샷이 아니라 `observation frame` 안에 들어간 제품 프리뷰를 둔다. 가장 적합한 장면은 Mine의 `Sector Scan` 장면이다. 이 섹션의 목적은 분위기만 보여주는 것이 아니라, 제품이 실제로 어떤 종류의 인터페이스인지 한눈에 이해시키는 데 있다.

### 4.3 Core Loop Preview

히어로 아래에는 `Mine → Vault → Lab`을 하나의 연결된 시스템으로 보여주는 섹션을 둔다. 세 단계는 평범한 기능 카드처럼 나열하지 않고, 하나의 신호선 또는 흐름 위에 배치된 시스템 노드처럼 구성한다.

각 노드는 아래 역할을 전달해야 한다.

- Mine: 가능성을 탐사하는 입구
- Vault: 가치 있는 결과를 축적하는 보관소
- Lab: 선택한 결과를 더 정교한 자산으로 발전시키는 공간

이 섹션의 목적은 IDEA MINE이 단순한 생성기가 아니라 발견, 선별, 발전의 루프를 가진 제품이라는 점을 짧고 명확하게 보여주는 데 있다.

### 4.4 Product / Interface Proof

이 섹션은 제품의 신뢰도를 보여주는 구간이다. 분위기만 강한 컨셉 페이지로 보이지 않게 하기 위해, 실제 앱 인터페이스의 밀도와 구조를 증명해야 한다.

구성은 2~3개의 실제 UI 프리뷰 패널로 잡는다.

- Mine 화면 일부
- Vault 화면 일부
- Lab 화면 일부

이 프리뷰는 소개 카드가 아니라, 작동하는 제품 패널을 잘라서 보여주는 느낌이어야 한다. 이 섹션의 목적은 "분위기 있는 제품"이 아니라 "실제로 일할 수 있는 제품"이라는 인상을 강화하는 데 있다.

### 4.5 Positioning Band

짧은 포지셔닝 섹션을 둔다. 이 구간은 브랜드 감정선보다 제품 정의를 다시 못 박는 역할을 한다.

예시 톤은 아래와 같다.

- 단순한 아이디어 생성 도구가 아님
- 발견, 선별, 정제를 위한 시스템

길게 설명하지 않고, 강한 문장과 짧은 보조 문장 정도로 구성한다.

### 4.6 Final CTA

랜딩의 마지막은 일반적인 마케팅 문구보다 시스템 안으로 진입하는 느낌을 주는 CTA 구간으로 마무리한다.

- 예: `Open the Observatory`, `Enter the Mine`, `Begin Signal Discovery`

이 구간은 다시 한 번 공간감을 살릴 수 있지만, 과장된 쇼케이스가 아니라 자연스럽게 앱 진입으로 이어져야 한다. 랜딩에서 가장 강한 핑크 반응은 이 지점과 히어로의 1차 CTA에만 제한적으로 허용한다.

### 4.7 Atmosphere Intensity by Section

랜딩 전체는 `Active Atmosphere`를 유지하되, 섹션별로 강도를 조절한다.

- Hero: 가장 깊고 시네마틱
- Core Loop: 시스템 구조가 더 잘 읽히도록 한 단계 절제
- Product Proof: UI가 전면에 오도록 분위기 강도 낮춤
- Final CTA: 마무리 장면으로서 우주감 다시 강화

## 5. Mine Page Wireframe

광맥 페이지의 역할은 `오늘의 광맥 신호를 관측하고, 세 개의 후보 중 하나를 선택해 탐사로 연결하는 대표 장면`이다. 이 페이지는 대시보드가 아니라 제품의 시그니처 씬이어야 한다.

### 5.1 Top Status Rail

광맥 페이지 상단에는 얇은 상태줄을 둔다.

- 좌측: 현재 공간 표시 (`Mine / Sector Scan`)
- 중앙: 스캔 상태와 운영 정보
- 우측: `re-scan`, 전역 이동 또는 프로필

예시 정보는 아래와 같다.

- `3 active veins`
- `1 deep scan available`
- `signal stability 92%`

이 영역은 정보성 HUD이므로 시안/스틸 계열을 중심으로 사용하고, 핑크는 거의 사용하지 않는다.

### 5.2 Central Sector Scan Field

페이지의 중심 무대는 넓은 `scan field`다. 평범한 콘텐츠 컨테이너가 아니라 인터랙션 가능한 관측 장면이어야 한다.

이 영역에는 아래 요소가 포함된다.

- 성운 배경
- 미세 입자
- 얇은 scan grid 또는 tracking line
- 느린 빛 흐름과 신호 반응
- 3개의 vein signal node

사용자는 페이지에 진입하자마자 "탐지된 광맥 신호 중 하나를 고르는 곳"이라는 사실을 직관적으로 이해해야 한다.

### 5.3 Three Asymmetric Signal Nodes

세 개의 광맥은 같은 무게의 카드로 나열하지 않는다. 서로 다른 위치와 거리감으로 배치된 `signal node`로 구성한다.

추천 배치는 아래와 같다.

- 하나는 중앙보다 약간 위
- 하나는 좌하단
- 하나는 우측 중간

이렇게 비대칭으로 배치해 장면을 더 덜 틀에 박히게 만든다.

각 노드는 아래 상태를 가진다.

- 기본 상태: 희미한 광물/신호 오브젝트
- hover 상태: 이름, 희귀도, 키워드 라벨 노출
- selected 상태: 링, 라인, 글로우, 연결선 강화

이 노드는 읽는 카드가 아니라 `관측 대상`이다. 텍스트는 항상 다 드러나지 않아도 되며, 선택될 때 더 선명해진다.

### 5.4 Selected Vein Detail Panel

선택된 광맥의 상세 정보는 별도 패널에 한 번에 하나만 강하게 보여준다. 추천 배치는 `우측 고정 분석 패널`이다. 이렇게 해야 중앙 무대를 넓게 유지하면서도 현재 선택과 세부 정보를 동시에 보여줄 수 있다.

이 패널에는 아래 정보가 들어간다.

- 광맥 이름
- 희귀도 또는 등급
- 조합 요소 2~4개
- 잠재력 요약 또는 짧은 설명
- 1차 CTA
- 보조 액션

패널의 재질감은 `instrument-heavy`가 기본이다. 대화창이나 일반 카드처럼 보여서는 안 되며, 분석 모듈처럼 안정적으로 느껴져야 한다.

### 5.5 CTA Hierarchy in Mine

이 페이지의 가장 중요한 행동은 선택한 광맥을 탐사로 연결하는 것이다. 따라서 CTA 위계는 매우 명확해야 한다.

- 1차 CTA: 선택된 광맥 패널 안의 메인 버튼
- 2차 CTA: `re-scan`
- 3차 액션: hover, preview, info reveal 등 보조 상호작용

가장 강한 핑크 반응은 선택된 광맥의 1차 CTA에만 준다. 나머지는 시안/스틸 기반 또는 신호 반응 수준으로 절제한다.

### 5.6 Support Blocks

메인 장면 아래나 우측 하단에 작은 보조 정보 블록 1~2개를 둘 수 있다.

예시:

- `How it works`
- `Signal stability`
- `Rarity distribution`

하지만 이 페이지는 정보 위주의 화면이 아니므로, 보조 블록은 작고 얌전해야 하며 중앙 무대와 상세 패널의 집중력을 해치면 안 된다.

### 5.7 Motion and State Change

광맥 페이지는 정적인 목록보다 상태 변화가 더 중요하다. 기본 모션 문법은 아래와 같다.

- 페이지 진입: scan field가 천천히 정렬됨
- node hover: 약한 링/선/라벨 반응
- node select: target lock 느낌의 강조
- detail panel update: 차분한 페이드/슬라이드
- CTA hover: 핑크 신호가 켜지는 느낌

모션은 시네마틱해야 하지만 느슨하면 안 된다. 사용자는 시스템이 반응하고 있다고 느껴야 한다.

## 6. Core Component Grammar

이번 리디자인에서 구현자가 바로 착수할 수 있도록 핵심 컴포넌트 규칙을 아래처럼 고정한다.

### 6.1 Status Rail

모든 주요 페이지 상단에 공통적으로 사용되는 얇은 상태줄이다.

- 역할: 브랜드 헤더가 아니라 시스템 HUD
- 구조: 좌측 상태, 중앙 정보, 우측 액션
- 반응: 시안/스틸 기반의 정밀한 hover 반응
- 금지: 두꺼운 마케팅 헤더, 강한 핑크 fill

### 6.2 Hero Panel

랜딩의 우측 프리뷰를 담는 관측 프레임이다.

- 역할: 제품이 실제로 어떻게 보이는지 보여주는 대표 장치
- 구조: 얇은 프레임, 상태 라벨, 약한 스캔 효과, 실제 UI 일부
- 금지: 단순 스크린샷 박스, 장식용 이미지 프레임

### 6.3 Signal Node

광맥 선택지를 구성하는 핵심 요소다.

- 역할: 카드가 아니라 탐지 대상
- 구조: 오브젝트, 라벨, 링, 연결선, hover 반응
- 상태: default / hover / selected
- 금지: 동일한 크기와 동일한 무게의 카드 3열

### 6.4 Detail Panel

선택된 광맥 정보를 보여주는 분석 모듈이다.

- 역할: 선택한 하나의 광맥에 대한 판단과 행동을 집중시킴
- 구조: 이름, 희귀도, 조합, 요약, CTA
- 성격: 안정적이고 읽기 쉬운 분석 패널
- 금지: 여러 광맥 정보 동시 노출, 대화창 같은 인상

### 6.5 CTA

CTA는 브랜드 색을 칠하는 도구가 아니라 신호 위계를 만드는 도구다.

- 기본: dark panel + steel border + pink signal accent
- Mine의 1차 CTA만 조금 더 강한 핑크 허용
- Vault/Lab 계열 CTA는 이번 문서 범위 밖이지만, 기본적으로 더 절제된 구조를 전제

### 6.6 Information Islands

읽어야 하는 콘텐츠는 항상 별도 정보 섬 안에 묶는다.

- 역할: 시네마틱한 배경 속에서도 가독성 확보
- 적용: core loop preview, product proof, detail panel, support blocks
- 금지: 배경 위에 직접 텍스트만 얹는 구성

### 6.7 Motion Grammar

모든 컴포넌트는 같은 상태 언어를 써야 한다.

- hover: 신호가 깨어나는 느낌
- selected: 타깃이 잠금되는 느낌
- transition: 레이어가 정렬되는 느낌
- 원칙: transform/opacity 중심, 과장보다 상태 전달 우선

## 7. CTA Hierarchy

CTA는 페이지마다 같은 규칙을 공유하되, 강도는 다르게 운용한다.

### 7.1 Landing

- 1차 CTA: 히어로와 최종 CTA 구간에서만 강한 진입 신호 사용
- 2차 CTA: 시스템 확인 또는 보조 탐색
- 원칙: 브랜드/진입을 강조하되, 마케팅 버튼처럼 과하게 칠하지 않음

### 7.2 Mine

- 1차 CTA: 선택된 광맥의 탐사 시작 버튼
- 2차 CTA: `re-scan`
- 3차 액션: 미리보기/상태 확인
- 원칙: 가장 강한 핑크는 `선택 완료 후 탐사 시작` 한 지점에만 집중

### 7.3 Pink Usage Rule

핑크는 CTA의 기본 fill이 아니라 `signal color`다. 따라서 아래처럼 사용한다.

- 자주 쓰는 위치: active border, selected state, hover glow, 핵심 CTA 강조
- 덜 쓰는 위치: 대형 fill, 전체 배경 조명, 과한 gradient

## 8. Implementation Readiness Notes

이번 문서는 구현 직전 기준으로 아래 결정만 고정한다.

- 이번 라운드 구현 대상은 `랜딩 + 광맥(Mine)` 두 페이지다.
- `Vault / Lab`의 본격 리디자인은 이번 문서 범위 밖이다.
- 상위 스타일 토큰은 `Premium-Space-UI-Style-Guide.md`의 `B+ Deep Relay` 팔레트와 CTA 규칙을 기준으로 한다.
- 픽셀 퍼펙트 시안 없이도 착수 가능하도록 페이지 구조, 정보 우선순위, 장면 역할, CTA 위계, 상태 변화를 결정 완료 상태로 정리한다.

이 문서를 읽은 구현자는 추가적인 방향 결정보다, 실제 레이아웃 구성과 컴포넌트 제작, 모션 구현, 반응형 정리에 집중할 수 있어야 한다.
