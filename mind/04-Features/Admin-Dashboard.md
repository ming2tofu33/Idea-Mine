# Admin Dashboard

> 앱 내 admin 전용 모니터링 대시보드. 별도 웹 어드민 없이, Camp 탭의 Admin 섹션에서 핵심 운영 지표를 실시간 확인.

---

## Admin 기능의 두 축

Camp 탭의 Admin 섹션은 **두 가지 성격의 기능**으로 구성:

| 축 | 문서 | 성격 |
|----|------|------|
| **Admin 설정** | [[Admin-Persona]] | 개발/테스트 도구 — 제한 해제, 페르소나, 시뮬레이션, 디버그 |
| **Admin 대시보드** | 이 문서 | 운영 모니터링 — KPI, 비용, 유저, Abuse, 시스템 |

---

## 설계 원칙

- **앱 내 통합** -- 별도 웹 어드민을 만들지 않음. 1인 개발에서 관리 포인트를 최소화
- **서버 사이드 보안** -- UI 노출 여부와 무관하게, 모든 admin 데이터는 백엔드에서 `role = 'admin'` 검증 후 제공
- **점진적 확장** -- Phase별로 필요한 패널만 추가. 처음부터 전부 만들지 않음

---

## Admin 섹션 구조

Camp 탭 > Admin 섹션 (role이 admin일 때만 노출)

```
Camp
├── 프로필 / 설정 (일반 유저)
└── Admin (admin 전용)
    │
    ├── [Admin 설정] ── 상세: Admin-Persona.md
    │   ├── 제한 해제          -- 무제한 모드, L1 해제, 일일 리셋
    │   ├── 페르소나 전환       -- Free/Lite/Pro + 제한 적용 토글
    │   ├── 프로필 조작         -- 레벨, 슬롯, 스트릭, 언어
    │   ├── 시뮬레이션          -- 광고, 구독, 날짜 점프, 키워드 지정
    │   ├── 디버그 도구         -- Daily State, AI 비용, Raw Response, 프롬프트
    │   ├── 테스트 데이터 관리   -- 금고/광맥/전체 초기화
    │   └── 환경 정보           -- 앱/백엔드 버전, Feature Flags
    │
    └── [Admin 대시보드] ── 상세: 이 문서 아래 섹션들
        ├── KPI 대시보드         -- 핵심 성공 지표
        ├── AI 비용 모니터링      -- API 비용 추적 + 알림
        ├── Abuse 모니터링       -- 악용 탐지 + 대응
        ├── 유저 관리            -- 유저 목록/상세/조치
        ├── 프롬프트 관리         -- 버전 관리 + A/B 테스트
        ├── 수익 현황            -- MRR, 전환율, churn
        ├── 광맥/키워드 관리      -- 핵심 콘텐츠 운영
        ├── AI 출력 품질          -- 생성 결과 샘플 + 품질 proxy
        ├── Feature Flags       -- 기능 on/off 토글
        ├── MCP 모니터링         -- 채널별 사용량/비용
        ├── 공지/이벤트           -- 앱 내 공지 + 이벤트 스케줄링
        └── 시스템 상태           -- API 헬스, 에러율, 앱 버전
```

## 2. KPI 대시보드

`KPI-Success-Metrics` 문서의 지표를 시각화.

### Phase 1 지표

| 지표 | 데이터 소스 | 시각화 |
|------|------------|--------|
| 첫 채굴 완료율 | analytics events | 퍼널 차트 |
| 원석 금고 반입률 | `vault_items` / `ideas` 비율 | 퍼센트 게이지 |
| 프로젝트 개요 생성률 | `project_outlines` / `vault_items` 비율 | 퍼센트 게이지 |
| D1 리텐션 | analytics events | 일별 라인 차트 |
| 유료 전환 클릭률 | analytics events | 퍼센트 + 추이 |

### MVP 행동 지표

| 지표 | 데이터 소스 |
|------|------------|
| 조합 1세트당 생성 실행률 | analytics events |
| 생성 후 저장률 | `vault_items` INSERT 추적 |
| 저장 후 재방문률 | 세션 로그 |
| 사용자당 평균 리롤 횟수 | `user_daily_state.rerolls_used` |

### 구현 방식

- Supabase에서 집계 뷰(materialized view) 또는 집계 테이블 생성
- 백엔드 admin 엔드포인트가 집계 데이터 반환
- 프론트엔드에서 간단한 차트 라이브러리로 시각화

## 3. AI 비용 모니터링

`ai_usage_logs` 테이블(13필드) 기반.

### 핵심 뷰

| 뷰 | 내용 |
|----|------|
| 일별 총 비용 | 날짜별 API 비용 합계 라인 차트 |
| 기능별 비용 | 채굴 / 개요서 / 감정 별 파이 차트 |
| 유저당 평균 비용 | 비용 분포 히스토그램 |
| 비용 이상치 | 평균 대비 3x 이상 유저 하이라이트 |
| L4 차단기 상태 | 일일/월간 비용 한도 대비 현재 사용량 게이지 |

### 알림

- 일일 비용이 임계값 초과 시 push 알림 (admin 디바이스)
- 월간 비용 예측이 예산의 80% 도달 시 경고

## 4. Abuse 모니터링

`Security-Policy`의 L1~L4 방어 체계와 연동.

### 핵심 뷰

| 뷰 | 내용 |
|----|------|
| Rate limit 히트 현황 | 시간대별 차단 횟수 |
| L3 행동 패턴 시그널 | 의심 패턴 로그 (빠른 반복 생성, 비정상 시간대 등) |
| 의심 유저 목록 | 시그널 점수 높은 유저 상위 N명 |
| 프롬프트 인젝션 시도 | 필터링된 입력 로그 |

### 조치

- Phase 1: 로그 확인만 (수동 판단)
- Phase 1.5~2: 의심 유저 일시 정지 / 경고 발송 기능 추가

## 8. 광맥/키워드 관리

IDEA MINE의 핵심 콘텐츠를 관리하는 패널. 키워드가 곧 제품의 원재료.

### 오늘의 광맥

| 기능 | 설명 |
|------|------|
| 현재 광맥 확인 | 오늘 노출 중인 광맥 3개 + 선정 근거 |
| 광맥 선정 로그 | 일별 광맥 히스토리 |
| 주말 특별 광맥 | 스케줄링 + 희귀 광맥 설정 |
| 실험적 광맥 | 노출 대상 (Lite/Pro) + 반응률 확인 |

### 키워드 통계

| 기능 | 설명 |
|------|------|
| 카테고리별 사용 빈도 | Who, Domain, Tech, Value, Money, AI 각각 인기 순위 |
| 커스텀 키워드 인기 순위 | 유저들이 직접 입력한 키워드 Top N |
| 조합 패턴 분석 | 자주 함께 쓰이는 키워드 조합 |
| 미사용 키워드 | 한 번도 선택되지 않는 키워드 목록 |

### 키워드 운영

| 기능 | 설명 |
|------|------|
| 키워드 추가/수정 | 앱 업데이트 없이 서버에서 키워드 관리 |
| 키워드 비활성화 | 삭제 대신 비활성화 (기존 데이터 보존) |
| AI 키워드 18개 관리 | Pro 유저가 선택하는 AI 기술 키워드 목록 관리 |

## 9. AI 출력 품질 모니터링

비용만 보면 "싸게 굴리고 있다"는 것만 알 수 있음. 품질까지 봐야 제품이 돌아가는지 판단 가능.

### 핵심 뷰

| 뷰 | 내용 |
|----|------|
| 최근 아이디어 샘플 | 최근 생성된 원석 10세트 열람 (AI가 이상한 출력을 하고 있진 않은지) |
| 최근 개요서 샘플 | 최근 프로젝트 개요서 N건 열람 |
| 최근 감정 샘플 | 최근 감정 결과 N건 열람 |
| 저장률 추이 | 일별 금고 반입률 -- 급락 시 품질 문제 신호 |
| 기능별 품질 proxy | 개요서 생성 후 재생성 비율 (높으면 첫 결과 불만족) |

### 품질 알림

- 저장률이 이동평균 대비 급락 시 알림
- 특정 키워드 조합에서 저장률이 유의미하게 낮으면 하이라이트

## 10. Feature Flags

앱 업데이트 없이 기능을 on/off. 1인 개발에서 "앱스토어 재심사 없는 롤백" 역할.

### 구조

```
feature_flags 테이블
├── flag_key: TEXT (예: 'full_combo_mode', 'experimental_vein')
├── enabled: BOOLEAN
├── target_tiers: TEXT[] (예: ['pro'], ['lite', 'pro'], ['all'])
├── description: TEXT
└── updated_at: TIMESTAMP
```

### 핵심 기능

| 기능 | 설명 |
|------|------|
| 플래그 목록 | 전체 feature flag 상태 한눈에 확인 |
| 즉시 토글 | on/off 전환, 대상 티어 변경 |
| 플래그 추가 | 새 기능 출시 시 flag 생성 |

### 활용 예시

- 풀 조합 모드 — Pro에서만 켜기, 문제 시 즉시 끄기
- 실험적 광맥 — Lite 이상에서 노출, 반응 안 좋으면 Pro로 축소
- 새 감정 타입 — 소수 유저에게만 먼저 오픈 (점진적 롤아웃)

## 기술 구현

### 아키텍처

```
앱 (Camp > Admin) --> Python 백엔드 (admin 엔드포인트) --> Supabase (집계 뷰)
                         ↓
                   role == 'admin' 미들웨어
```

### 백엔드 엔드포인트 구조

```
/api/admin/
├── persona/          -- 페르소나 전환
├── kpi/              -- KPI 집계 데이터
├── costs/            -- AI 비용 집계
├── abuse/            -- Abuse 로그 + 시그널
├── users/            -- 유저 목록/상세
├── prompts/          -- 프롬프트 버전 관리
├── revenue/          -- 수익 지표 (Phase 1.5+)
├── veins/            -- 광맥 관리 + 키워드 통계
├── keywords/         -- 키워드 CRUD + 커스텀 키워드 통계
├── quality/          -- AI 출력 샘플 + 품질 지표
├── flags/            -- Feature flag CRUD + 토글
├── mcp/              -- MCP source별 사용량/비용
├── notices/          -- 공지사항 CRUD
├── events/           -- 이벤트 스케줄링
└── health/           -- 시스템 상태 집계
```

### 보안

- 모든 `/api/admin/*` 엔드포인트에 admin 미들웨어 적용
- RLS: 집계 테이블/뷰는 admin role만 SELECT 허용
- 상세 보안 정책은 [[Security-Policy]] 참조

### DB 설계 방향

- `admin_settings` 테이블: admin 계정별 설정 저장 (페르소나 tier, 대시보드 환경설정 등)
- `feature_flags` 테이블: flag_key, enabled, target_tiers, description, updated_at
- `notices` 테이블: 공지사항 CRUD + 예약 발행 + 만료
- 집계 뷰: KPI/비용/유저 통계를 위한 materialized view (실시간 쿼리 부담 줄임)
- 별도 집계 테이블 vs materialized view는 데이터 규모에 따라 결정

## Phase별 구현 범위

| Phase | 패널 | 이유 |
|-------|------|------|
| Phase 1 | AI 비용 모니터링, Abuse 로그, 유저 목록 (최소), AI 출력 품질 (샘플 열람), 시스템 상태, Feature Flags | MVP 운영에 필수인 비용/품질/안정성 모니터링 |
