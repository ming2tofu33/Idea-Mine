# The Mine 홈 화면 와이어프레임

> 큰 `채굴 시작` CTA를 중심으로, 대시보드형 정보성과 탐험형 분위기를 함께 살리는 홈 화면 설계.

## 결정 사항

- 홈 화면의 성격은 `대시보드형 + 탐험형` 하이브리드
- 첫 시선은 `큰 채굴 시작 CTA`와 `오늘의 광맥`으로 유도
- 광맥 3개는 처음에 `동등하게` 노출
- 사용자가 흥미로운 광맥을 직접 탭하면 해당 광맥만 강조/확장
- CTA는 홈 하단에 항상 보이되, 선택 전/후 상태 문구를 다르게 운영
- CTA를 눌렀을 때 바로 채굴하지 않고, 선택된 광맥을 바탕으로 짧은 채굴 연출 후 결과 화면으로 이동

---

## 홈 화면 목표

- 앱에 들어오자마자 "오늘 뭘 캘지"가 분명히 보이게 한다
- 정보는 충분히 제공하되, 홈이 대시보드처럼 딱딱해지지 않게 한다
- 유저가 직접 광맥을 고르는 재미를 만든다
- `채굴 시작` CTA가 가장 중요한 행동으로 자연스럽게 이어지도록 만든다

---

## 화면 구조

### 1. 상단 상태 바

아주 얇은 정보 영역. 홈의 주인공은 아니지만, 오늘의 진행 상태를 빠르게 이해하게 해준다.

- 연속 기록
- 오늘 채굴 가능 횟수
- 보유 재화

예시:

```text
[연속 3일]   [오늘 1/1]   [코인 120]
```

### 2. 홈 헤더

세계관과 행동을 함께 보여주는 짧은 영역.

- 타이틀: `The Mine`
- 서브카피: `오늘은 어떤 광맥을 캐볼까요?`

설명은 짧고 직관적으로 유지한다. 긴 소개 문구는 넣지 않는다.

### 3. 오늘의 광맥 영역

홈 화면의 메인 콘텐츠. 광맥 3개를 처음엔 동등하게 보여준다.

- 화면 중단에 크게 노출
- 카드 크기는 충분히 커서 "고르는 재미"가 느껴져야 함
- 가로 슬라이드보다 세로 스택형이 적합
- 우측 상단에 작은 `다시 섞기` 액션 배치 가능

각 카드에 들어갈 정보:

- 광맥 이름
- 희귀도 또는 타입
- 분위기 설명 1줄
- 키워드 칩 2~3개
- 작은 시각 요소(원석, 광맥 아이콘, 픽셀 장식)

### 4. 선택 후 확장 영역

광맥을 하나 탭하면 해당 카드만 강조되고, 그 카드 내부 또는 바로 아래에 상세 영역이 열린다.

- 선택된 카드만 골드 프레임/광택/그림자 등으로 강조
- 나머지 2개 카드는 축약 상태 유지
- 확장 영역에는 "이 광맥에서 어떤 아이디어가 나올지" 기대감을 주는 내용 배치

권장 정보:

- 선택한 광맥의 한 줄 설명
- 예상 아이디어 톤 2~3줄
- 확장된 키워드 칩 4~6개
- 추천 이유 또는 짧은 힌트

### 5. 하단 고정 CTA

홈 화면의 핵심 액션. 스크롤과 무관하게 하단에 항상 보인다.

- 선택 전: `광맥을 선택하고 채굴 시작하기`
- 선택 후: `OO 광맥 채굴 시작하기`

선택 전에도 버튼을 숨기지 않고 존재를 유지한다. 다만 선택 유도형 텍스트와 약한 상태를 사용한다.

## CTA를 눌렀을 때의 흐름

홈에서는 광맥 선택과 기대감 형성에 집중하고, 실제 채굴 시작은 짧은 연출을 거쳐 결과 화면으로 전환한다.

```text
[반짝 협곡 채굴 시작하기] 탭
        ↓
짧은 채굴/스캔 연출
        ↓
"광맥을 스캔하는 중..."
"아이디어 결정을 추출하는 중..."
        ↓
결과 화면 진입
```

이 흐름의 장점:

- 버튼을 눌렀을 때 행동의 무게감이 생김
- 선택한 광맥과 결과 화면 사이의 연결감이 강해짐
- 채굴 세계관을 체감하게 해줌
- 단순 로딩보다 훨씬 제품 개성이 살아남

---

## 인터랙션 원칙

- 광맥 3개는 처음에 모두 같은 위계로 노출
- 사용자가 탭한 광맥만 선택 상태로 전환
- 선택 상태는 컬러, 테두리, 확장 높이 변화로 분명하게 표현
- `다시 섞기`는 보조 액션으로만 취급
- 홈에 정보는 넣되, 카드/지표가 광맥보다 더 눈에 띄어선 안 됨

---

## 카피 방향

- 헤더 카피는 짧고 행동 유도형
- 광맥 설명은 "무엇이 나올지" 상상되게 쓰기
- CTA는 항상 동사 중심으로 명확하게 쓰기

예시:

- `오늘은 어떤 광맥을 캐볼까요?`
- `사람들의 반응을 끌어내는 흥미로운 아이디어 광맥`
- `광맥을 선택하고 채굴 시작하기`
- `반짝 협곡 채굴 시작하기`

---

## 왜 이 방향이 좋은가

- 홈을 열자마자 앱의 핵심 아이디어인 `광맥 선택`이 드러난다
- 대시보드 정보와 탐험 감성을 동시에 유지할 수 있다
- 유저가 직접 선택하기 때문에 몰입감과 소유감이 커진다
- 큰 CTA가 살아 있으면서도, 선택의 맥락이 분명하다

---

## 구현 시 체크 포인트

- 카드 3개가 모바일 화면에서 충분히 크게 보여야 한다
- 선택 전과 선택 후의 상태 차이가 즉시 느껴져야 한다
- 하단 CTA는 항상 안정적으로 고정되어야 한다
- 채굴 시작 후 짧은 전환 연출은 과하지 않게 1~2초 내외로 유지
- 홈 화면이 과도하게 복잡해지지 않도록 보조 정보는 최소화한다
