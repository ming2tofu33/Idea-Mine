# Camp 화면 방향 정리

> `Camp`를 프로필 허브가 아니라 `광산에서 돌아온 뒤 머무는 작은 야영지`로 재정의한 메모.
> 이 문서는 [[2026-03-22-camp-screen]]의 시각 방향을 보정하고, 이후 UI 구현의 기준점으로 사용한다.

> [!summary]
> Camp의 1차 역할은 `정보 관리`가 아니라 `내 공간 체감`이다.
> 화면은 `작은 야영지 + 캠프파이어`가 중심이 되며, 프로필 정보는 카드가 아니라 공간 오브젝트에 녹여서 보여준다.

## 왜 방향을 바꾸는가

- 기존 Camp 초안은 프로필 카드, 활동 기록, 설정 메뉴를 중심에 두고 있었다.
- 하지만 [[The-Mine]]의 역할은 `오늘 무엇을 할지 바로 고르게 하는 행동 중심 화면`이고, Camp는 그와 다른 감정과 역할을 가져야 한다.
- Camp까지 프로필 허브처럼 만들면 `Mine`과 정보 성격이 겹치고, Camp의 존재 이유가 약해진다.
- 따라서 `Mine = 행동 중심`, `Camp = 정체성/분위기 중심`으로 분리하는 편이 더 맞다.

## 역할 분리 원칙

### The Mine

- 오늘의 채굴 동선
- 빠른 상태 확인
- 즉시 행동 유도

Mine에는 아래 정보가 어울린다.

- 오늘 채굴 가능 횟수
- 연속 기록
- 보유 재화
- 최근 성과 요약

### Camp

- 광산에서 돌아온 광부의 자리
- 내 흔적과 분위기를 보여주는 공간
- 나중에 꾸미기와 확장으로 이어질 수 있는 기반

Camp에는 아래 정보만 남긴다.

- 닉네임
- 광부 레벨
- 대표 상징 1개
- 상태 문구 한 줄
- 설정 진입

## 감정 톤

Camp의 핵심 감정은 아래 조합으로 잡는다.

- 메인 감정: `소박하지만 성취감 있는 감정`
- 보조 감정: `따뜻한 휴식감`

즉, `아늑한 쉼터`가 1순위라기보다 `오늘도 하나 캐고 돌아온 자리`가 먼저 느껴져야 한다.

이 방향은 다음 노트들과도 잘 맞는다.

- [[Tone-&-Manner]]
- [[Decoration-System]]

## 핵심 장면

Camp의 첫 장면은 아래 구도로 고정한다.

- 캠프파이어 앞 작은 텐트
- 옆에 상자 또는 전리품 더미
- 나무 표지판에 닉네임과 레벨
- 랜턴 1개
- 멀리 보이는 광산 입구 실루엣

핵심은 `캠프 장면 자체가 메인 콘텐츠`라는 점이다.
프로필 카드가 장면 위에 떠 있는 구조가 아니라, 장면 안에 정보가 녹아 있어야 한다.

## 오브젝트별 역할

### 캠프파이어

- 화면에서 가장 밝은 포인트
- 감정 중심 오브젝트
- 작은 불꽃 애니메이션으로 생동감 부여
- Phase 1에서는 기능 버튼이 아니라 분위기 중심 요소로 유지

### 텐트

- 광부의 개인 공간
- Camp를 `내 자리`로 느끼게 하는 핵심 배경 오브젝트
- Phase 2 이후 `꾸미기` 진입점으로 확장 가능

### 나무 표지판

- 닉네임과 광부 레벨 노출
- 별도 프로필 카드 대신 정체성을 드러내는 장치
- 예시: `도민 광부 Lv.3`

### 상자 또는 전리품 더미

- 대표 배지 1개 또는 최근 성취를 암시하는 오브젝트
- 숫자 나열보다 `작게 전시된 결과물`로 보여주는 쪽이 적합

### 랜턴

- 보조 광원
- 텐트와 표지판 사이 시선 흐름 정리
- 화면이 지나치게 비어 보이지 않도록 균형 보강

### 멀리 보이는 광산 입구

- [[The-Mine]]과 Camp를 이어주는 배경 장치
- 전경보다 튀지 않게 실루엣 수준으로만 처리

## Camp 메인 화면에 남길 정보

- 닉네임
- 광부 레벨
- 대표 배지 1개
- 상태 문구 1줄
- 설정 버튼

### 상태 문구 예시

- `오늘도 원석 하나를 가져왔어요`
- `모닥불 옆에 오늘의 전리품을 내려두었습니다`
- `캠프가 조금씩 광부의 흔적으로 채워지고 있어요`

## Camp 메인 화면에서 뺄 정보

- 이메일
- 총 채굴 수
- 총 원석 수
- 보유 재화 전체 수치
- 연속 접속 수치 칩
- 긴 활동 기록 리스트
- 구독 상태
- 로그아웃 버튼 메인 노출
- 배지 전체 목록
- 관리자 정보

기준은 단순하다.

> [!note]
> Camp 메인에 뜨는 정보는 `이 사람이 누구인가`, `이 공간의 분위기가 무엇인가`, `최근에 무엇을 해냈는가` 중 하나는 설명해야 한다.
> 이 셋에 들어가지 않으면 메인에서는 뺀다.

## 화면 배치 와이어프레임

```text
┌──────────────────────────────────┐
│  Camp                      [⚙]   │
│    ·      ·        ·             │
│         밤하늘 / 별빛             │
│   멀리 광산 입구 실루엣           │
│                                  │
│  ┌────────────┐        [랜턴]     │
│  │ 도민 광부  │          │        │
│  │   Lv.3     │                   │
│  └────────────┘      ┌────────┐   │
│                      │ 텐트   │   │
│         광부 캐릭터   └────────┘   │
│            \o/          [상자]    │
│             |           [배지]    │
│         (( 캠프파이어 ))          │
│                                  │
│──────────── 흙바닥 / 풀밭 ────────│
│   오늘도 원석 하나를 가져왔어요    │
│                                  │
│   [배지 보기]        [꾸미기 Soon] │
└──────────────────────────────────┘
```

## 레이아웃 원칙

- 화면 중심은 항상 캠프파이어가 차지한다.
- 텐트는 우측 뒤에 살짝 물려 배치해 개인 공간처럼 보이게 한다.
- 나무 표지판은 좌측 전경에 두고, 카드 대신 이름과 레벨만 보여준다.
- 상자와 대표 배지는 우측 하단 전경에 두어 `최근 성취`를 암시한다.
- 하단 영역은 두꺼운 메뉴 바 대신 얇은 도크만 둔다.
- 설정은 감성 오브젝트와 섞지 않고 우측 상단 시스템 버튼으로 분리한다.

## 인터랙션 원칙

- Phase 1에서 즉시 눌리는 것은 `설정`, `배지 보기` 정도만 허용
- `꾸미기`는 비활성 또는 `Soon` 처리
- 텐트, 캠프파이어, 광산 입구는 우선 장면용 오브젝트로 유지
- 초반부터 모든 오브젝트를 메뉴화하지 않는다

Camp는 `메뉴 허브`가 아니라 `보는 재미가 먼저인 공간`이어야 한다.

## 성장 방향

Camp는 고정된 배경이 아니라 `작은 야영지 -> 정착 캠프 -> 베이스`로 성장할 수 있어야 한다.

### Phase 1

- 작은 캠프파이어 야영지
- 공간감과 정체성만 전달

### Phase 2

- 장비가 늘어난 정착 캠프
- 오브젝트 단위 꾸미기 확장

### Phase 3

- 오두막 또는 베이스 느낌의 상위 테마
- 실내/작업 공간 요소 일부 확장 가능

핵심은 `교체`가 아니라 `성장`이다.
갑자기 전혀 다른 장르의 방 꾸미기 화면으로 바뀌면 세계관 연결감이 약해질 수 있다.

## 구현 메모

- 기존 `프로필 카드 중심` Camp 구조는 유지하지 않는다.
- `Mine`에는 얇은 상태 정보 바를 두고, Camp에는 정체성/분위기 정보를 남긴다.
- Camp는 설명하는 화면보다 `보여주는 화면`이어야 한다.
