# 로딩 애니메이션 설계

> 모든 로딩 상황에 세계관에 맞는 고유 연출을 적용한다.
> 기본 스피너(ActivityIndicator)는 사용하지 않고, 각 행위의 의미에 맞는 픽셀 스타일 애니메이션을 구현한다.

> [!summary]
> 7개 로딩 상황마다 전부 다른 연출. 짧은 로딩(1~2초)은 가볍고 빠르게, 긴 로딩(5초+)은 단계적 진행감으로 설계.
> 모든 연출은 코드 기반(Animated API). 픽셀 에셋은 Phase 2에서 교체.

---

## 설계 원칙

- **세계관 일관성**: 모든 로딩은 광산 세계관의 행위에 대응한다
- **시간 비례 복잡도**: 짧은 로딩은 단순하게, 긴 로딩은 단계적으로
- **정직한 진행감**: 가짜 진행률 바 대신, 행위 자체의 변화로 진행을 보여준다
- **Phase 1은 코드 + 이모지**: 에셋 없이 View, Animated, 이모지로 구현. Phase 2에서 스프라이트 교체

## 1. 앱 시작 — 광산 입장

> 세계관: 광부가 랜턴을 들고 광산에 입장하는 순간
> 위치: `_layout.tsx` (세션 확인 동안)
> 대기 시간: 1~2초

### 연출 흐름

```
[0.0초] 완전 어두운 화면 (#000000)
        아무것도 보이지 않는다

[0.5초] 화면 정중앙에 랜턴 이모지(🏮)가 나타남
        랜턴을 중심으로 작은 원형 glow가 생김
        glow 색상: rgba(196,176,122,0.15) — accent.goldGlow
        glow 크기: 0 → 60px 반경으로 서서히 확장

[1.0초] glow가 더 넓게 퍼짐 (60 → 160px)
        퍼지는 빛 안에서 "IDEA MINE" 로고가 골드(accent.gold)로 드러남
        로고는 opacity 0 → 1 페이드인
        폰트: Galmuri11-Bold, 28px

[1.2초] 로고 아래에 텍스트 페이드인
        "광산에 입장하는 중..."
        폰트: Galmuri11, caption 스타일, text.muted 색상

[로딩 완료] 전체 화면이 서서히 밝아지면서 메인 화면으로 전환
```

### 애니메이션 상세

| 요소 | 속성 | 시작 | 끝 | 시간 | 이징 |
|------|------|------|-----|------|------|
| 배경 | backgroundColor | #000000 | 유지 | - | - |
| 랜턴 | opacity | 0 | 1 | 0.3초 | ease-in |
| glow 1차 | scale | 0 | 1 | 0.5초 | ease-out |
| glow 2차 | scale | 1 | 2.5 | 0.5초 | ease-out |
| 로고 | opacity | 0 | 1 | 0.3초 | ease-in |
| 서브텍스트 | opacity | 0 | 1 | 0.3초 | ease-in |

### 구현 노트

- glow는 `borderRadius: 999` 원형 View + `backgroundColor: goldGlow` + scale 애니메이션
- 랜턴은 Phase 1에서 이모지(🏮), Phase 2에서 `lantern.png` 스프라이트
- 로딩 완료 시 화면 전환은 Expo Router의 자연스러운 전환에 맡김
- 스플래시 스크린(expo-splash-screen)과 연결해서 네이티브 스플래시 → 이 연출 → 메인 화면 흐름

## 2. 광맥 불러오기 — 랜턴으로 벽 탐색

> 세계관: 광부가 랜턴을 들고 광산 벽을 비추며 오늘의 광맥을 찾는 중
> 위치: `index.tsx` (광산 홈, veins 로딩 동안)
> 대기 시간: 1~2초

### 연출 흐름

```
[시작] 광산 홈 화면의 광맥 카드 영역에 표시

        랜턴 불빛(타원형 glow)이 왼쪽 벽에서 시작
        "벽을 살피는 중..."

[0.5초] 불빛이 왼쪽 → 중앙으로 이동
        이동 경로를 따라 벽면이 은은하게 밝아짐

[1.0초] 불빛이 중앙 → 오른쪽으로 이동

[1.5초] 불빛이 오른쪽 끝에 도달하면 다시 왼쪽으로
        로딩 완료될 때까지 반복

[완료]  불빛이 멈추고, 광맥 카드 3개가 페이드인으로 등장
```

### 애니메이션 상세

| 요소 | 속성 | 동작 | 시간 | 이징 |
|------|------|------|------|------|
| 랜턴 불빛 | translateX | 좌(0) → 우(화면폭) | 1.5초 | ease-in-out |
| 불빛 glow | opacity | 0.3 → 0.6 → 0.3 | 불빛 위치에 따라 | linear |
| 벽면 밝기 | opacity | 불빛 근처만 밝아짐 | 불빛 따라감 | - |

### 구현 노트

- 랜턴 불빛은 타원형 View (`width: 80, height: 120, borderRadius`) + goldGlow 배경
- translateX 애니메이션으로 좌→우 반복 이동
- 실제 벽면 밝기는 불빛 View의 glow가 자연스럽게 처리
- 광맥 카드 영역 전체를 덮는 오버레이로 구현
- 로딩 완료 시 오버레이 fadeOut → 카드 fadeIn

## 3. 채굴 중 — 곡괭이로 광맥 캐기 (구현 완료)

> 세계관: 광부가 곡괭이로 광맥을 내리쳐서 원석을 추출하는 중
> 위치: `MiningLoader.tsx` (전체 화면)
> 대기 시간: 5~15초

### 연출 흐름

```
[1단계: 스캔] 4초
        곡괭이(⛏)가 반복적으로 내리침
        내리칠 때마다 골드 파편(✦ ✦) 튀어오름
        광맥 블록: 큰 크기 (64x48)
        광차에 원석 💎 1개
        "광맥을 스캔하는 중..."

[2단계: 추출] 4초
        곡괭이 계속 내리침
        광맥 블록: 중간 크기 (48x40) — 캐지는 중
        광차에 원석 💎💎 2개
        "결정 구조를 분석하는 중..."

[3단계: 완료 직전] 4초~
        곡괭이 계속
        광맥 블록: 작은 크기 (32x32) — 거의 다 캠
        광차에 원석 💎💎💎 3개
        "아이디어 결정을 추출하는 중..."
```

### 곡괭이 애니메이션 사이클 (1.35초)

```
[0.0초] 대기 위치 (0도)
[0.4초] 들어올리기 (-45도) — ease-out
[0.6초] 내리치기 (+15도) — ease-in (빠르게)
[0.75초] 반동 (0도)
[1.35초] 쉬기 (0.6초 대기)
→ 반복
```

### 파편 타이밍

- 곡괭이가 내리치는 순간(0.6초)에 맞춰 파편 등장
- opacity 0→1 (0.1초) → 위로 떠오르며 사라짐 (0.3초)
- 곡괭이 사이클과 동기화

### 구현 상태

- 구현 완료: `components/mine/MiningLoader.tsx`
- Phase 2 교체: 이모지 → 스프라이트 애니메이션 (곡괭이 스윙 4프레임, 파편 스프라이트)

## 4. 리롤 — 광맥 폭파 교체

> 세계관: 기존 광맥을 다이너마이트로 폭파하고, 새 벽면에서 새 광맥을 찾는 중
> 위치: `index.tsx` (광맥 카드 영역, 인라인)
> 대기 시간: 1~2초

### 연출 흐름

```
[0.0초] 리롤 버튼 누름
        기존 광맥 카드 3개가 짧게 흔들림 (좌우 2px, 3회)
        진동/떨림 느낌

[0.3초] 카드들이 흔들림 멈추고 살짝 축소 (scale 0.95)

[0.4초] 💥 폭발 이펙트
        카드 위치에 골드+오렌지 파편이 사방으로 튐
        기존 카드 opacity → 0 (순간적으로 사라짐)
        짧은 Haptic 피드백 (이미 구현됨)

[0.6초] 먼지 이펙트
        회색 연기/먼지가 퍼지는 느낌
        opacity 0.6 → 0 으로 서서히 걷힘
        색상: text.muted 계열

[1.0초] 먼지가 걷히면서 새 광맥 카드 3개가 등장
        scale 0.8 → 1.0 + opacity 0 → 1
        벽에서 솟아오르는 느낌

[1.2초] 새 카드 완전히 안착. 선택 가능 상태
```

### 애니메이션 상세

| 요소 | 속성 | 동작 | 시간 | 이징 |
|------|------|------|------|------|
| 기존 카드 | translateX | 0 → ±2 → 0 (3회) | 0.3초 | linear |
| 기존 카드 | scale + opacity | 1→0.95→0 | 0.15초 | ease-in |
| 폭발 파편 | scale + opacity | 0→1→0 | 0.3초 | ease-out |
| 먼지 | opacity | 0.6→0 | 0.4초 | ease-out |
| 새 카드 | scale + opacity | 0.8→1 + 0→1 | 0.3초 | ease-out (bounce) |

### 구현 노트

- 폭발 파편은 4~6개의 작은 View (4x4px, 골드/오렌지)가 중심에서 사방으로 퍼짐
- 먼지는 큰 원형 View (borderRadius) + text.muted 배경 + opacity 애니메이션
- 전체 연출이 1.2초로 짧아서 빠른 느낌 유지
- Haptic은 폭발 시점(0.4초)에 `impactAsync(Heavy)`

## 5. 금고 반입 — 광차 출발

> 세계관: 선택한 원석을 광차에 실어서 레일을 타고 금고로 운반하는 중
> 위치: `mining-result.tsx` (반입 버튼 영역 또는 오버레이)
> 대기 시간: 1~2초

### 연출 흐름

```
[0.0초] "금고로 반입하기" 버튼 누름
        화면 하단에 레일이 나타남 (왼→오 직선)
        레일 색상: #4A3728 (나무 레일)

[0.2초] 선택한 원석들이 하나씩 아래로 떨어져서 광차에 담김
        원석마다 약간의 딜레이 (stagger 0.1초)
        담길 때 작은 바운스 효과
        광차는 화면 왼쪽에 위치

[0.6초] 광차가 레일을 타고 오른쪽으로 출발
        처음에 느리게 → 점점 빨라짐 (ease-in)
        광차 위의 원석들이 약간 흔들림 (덜컹덜컹)

[1.0초] 광차가 화면 오른쪽 밖으로 사라짐

[1.2초] "금고에 안전하게 보관했어요" 텍스트 페이드인
        또는 바로 화면 전환
```

### 광차 구조

```
        💎💎💎    ← 선택한 원석들
      ┌────────┐
      │  광차  │    ← 나무색 본체 (#4A3728)
      └────────┘
       ●      ●    ← 바퀴
  ═══════════════   ← 레일
```

### 애니메이션 상세

| 요소 | 속성 | 동작 | 시간 | 이징 |
|------|------|------|------|------|
| 레일 | opacity | 0→1 | 0.2초 | ease-in |
| 원석 낙하 | translateY | -20→0 | 0.15초/개 | bounce |
| 광차 이동 | translateX | 0 → 화면폭+100 | 0.6초 | ease-in (가속) |
| 원석 흔들림 | translateY | 0→-2→0 | 0.1초 반복 | linear |
| 완료 텍스트 | opacity | 0→1 | 0.3초 | ease-in |

### 구현 노트

- 광차는 MiningLoader의 Minecart 컴포넌트 재사용 가능
- 원석 개수는 실제 선택한 개수에 맞춤
- 레일은 화면 하단에 가로 직선 View
- 광차 이동 중 원석 흔들림은 작은 translateY 반복으로 "덜컹거리는" 느낌
- 화면 밖으로 사라진 후 API 응답 대기 → 완료 텍스트 또는 화면 전환

## 6. 개요서 생성 — 원석 연마 + 개요서 틀 등장

> 세계관: 실험실에서 거친 원석을 다듬어 빛나는 보석으로 만들고, 그 보석을 프로젝트 개요서로 정리하는 중
> 위치: `lab.tsx` (전체 화면 전환)
> 대기 시간: 20~40초

### Phase A: 원석 연마 (0~8초)

실험실 장치가 작동하면서 원석이 점점 다듬어지는 단계.

```
[0.0초] 민트 블루 배경의 작업대 위에 거친 원석 배치
        원석은 각진 형태 (거친 사각형 View)
        색상: 어두운 회색 (#4A4E62)
        유리관에 작은 거품 오르기 시작
        램프 점등
        "원석의 결을 정리하는 중이에요"

[3.0초] 원석이 살짝 회전하며 모서리가 깎임
        형태: 각진 사각형 → 둥근 사각형 (borderRadius 증가)
        색상: 회색 → 약간 밝아짐 (#5A5E72)
        유리관 거품 빨라짐
        "표면의 불순물을 제거하는 중이에요"

[6.0초] 원석이 한 번 더 다듬어짐
        형태: 둥근 사각형 → 다이아몬드형 (45도 회전 + 축소)
        색상: 밝은 색으로 전환 (#A8E6CF — 민트)
        glow 효과 시작
        "결정의 내면이 드러나고 있어요"

[8.0초] 원석이 완전히 빛나는 보석으로 변환
        형태: 다이아몬드형 유지 + 강한 glow
        색상: accent.gold 또는 lab.panel.default
        "✨ 빛나는 원석이 됐어요"
```

### Phase A 원석 변화 상세

```
[거친 원석]     [1차 연마]      [2차 연마]      [완성 보석]
 ┌──────┐      ╭──────╮       ◇              ◆ ✨
 │▓▓▓▓▓▓│      │▒▒▒▒▒▒│      ╱ ░░ ╲          glow
 │▓▓▓▓▓▓│  →   │▒▒▒▒▒▒│  →  ╲ ░░ ╱    →     ◆ ✨
 └──────┘      ╰──────╯       ◇              glow
  #4A4E62       #5A5E72      #A8E6CF        accent.gold
```

| 단계 | borderRadius | rotation | scale | 색상 | glow |
|------|-------------|----------|-------|------|------|
| 거친 원석 | 0 | 0deg | 1.0 | #4A4E62 | 없음 |
| 1차 연마 | 8 | 0deg | 0.95 | #5A5E72 | 없음 |
| 2차 연마 | 4 | 45deg | 0.7 | #A8E6CF | 약함 |
| 완성 보석 | 4 | 45deg | 0.7 | accent.gold | 강함 |

### Phase B: 전환 (8~10초)

연마 완료에서 개요서 작성으로 넘어가는 순간.

```
[8.0초] 완성된 보석이 살짝 위로 떠오름 (translateY -20)
        "이제 문서로 정리할게요"

[9.0초] 보석이 축소되며 화면 상단으로 이동
        작업대와 소품들이 fadeOut
        배경이 실험실 씬 → 문서 배경으로 부드럽게 전환

[10.0초] 개요서 틀이 아래에서 올라옴
```

### Phase C: 개요서 작성 (10초~완료)

빈 개요서 틀이 순서대로 채워지는 단계. 가장 긴 대기 구간.

```
[10.0초] 개요서 카드 틀 등장 (아래에서 slideUp)
         상단에 작은 보석 아이콘 + 원석 이름
         "프로젝트 개요 틀을 맞추고 있어요"

[13.0초] 첫 번째 섹션 등장: "문제 정의"
         섹션 제목만 먼저 나타남
         내용 영역은 점선 또는 shimmer 로딩
         "문제를 정의하는 중이에요"

[17.0초] 두 번째 섹션 등장: "타깃 사용자"
         "타깃 사용자를 분석하는 중이에요"

[21.0초] 세 번째 섹션 등장: "핵심 기능"
```

## 7. 로그인 — 출입증 스캔

> 세계관: 광산 입구에서 광부 출입증을 스캔하고 신원을 확인하는 중
> 위치: `sign-in.tsx` (OAuth 처리 동안)
> 대기 시간: 2~3초

### 연출 흐름

```
[0.0초] 로그인 버튼 누름
        버튼 영역이 출입증 카드로 변환
        카드 형태: 직사각형, 어두운 배경 + 골드 테두리
        카드 내용:
          상단: "IDEA MINE" 작은 로고
          중앙: 광부 이모지 (⛏) 또는 실루엣
          하단: "광부 출입증"

[0.3초] 스캔 라인 시작
        밝은 골드 가로 라인 (2px 높이)이 카드 위에서 아래로 내려감
        스캔 라인 색상: accent.gold
        라인 뒤에 약간의 glow 잔상

[0.8초] 스캔 라인이 카드 하단에 도달
        다시 위로 올라감
        "광부 신원 확인 중..."

[1.3초] 두 번째 스캔 (위→아래)
        반복

[인증 완료] 스캔 라인 멈춤
            카드 테두리가 골드로 밝게 빛남 (glow 강화)
            "✓ 입장 허가" 텍스트
            0.5초 후 메인 화면으로 전환
```

### 출입증 카드 구조

```
┌─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┐
│                          │
│      IDEA MINE           │
│                          │
│         ⛏               │
│      [광부 실루엣]        │
│                          │
│ ═══════════════════════  │  ← 스캔 라인 (골드, 위→아래 이동)
│                          │
│      광부 출입증          │
│                          │
└─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┘
     골드 테두리 (2px)

"광부 신원 확인 중..."
```

### 애니메이션 상세

| 요소 | 속성 | 동작 | 시간 | 이징 |
|------|------|------|------|------|
| 버튼→카드 전환 | scale + borderRadius | 버튼 크기 → 카드 크기 | 0.3초 | ease-out |
| 스캔 라인 | translateY | 0 → 카드높이 | 0.5초 | linear |
| 스캔 라인 | translateY | 카드높이 → 0 | 0.5초 | linear |
| 스캔 glow 잔상 | opacity | 0.8 → 0 | 0.2초 | ease-out |
| 완료 테두리 glow | shadowRadius | 0 → 12 | 0.3초 | ease-out |
| 완료 텍스트 | opacity | 0 → 1 | 0.2초 | ease-in |

### 구현 노트

- 로그인 버튼이 카드로 변환되는 연출은 해당 버튼만 교체 (다른 버튼은 숨김)
- 스캔 라인은 절대 위치 View + translateY 반복 애니메이션
- glow 잔상은 스캔 라인 뒤에 따라오는 반투명 View (opacity 빠르게 감소)
- OAuth 리다이렉트가 시작되면 앱이 백그라운드로 가므로, 돌아올 때의 상태도 고려
  - OAuth에서 돌아왔을 때: 스캔 완료 연출 → 메인 화면
  - OAuth에서 취소했을 때: 카드 → 버튼으로 되돌림
