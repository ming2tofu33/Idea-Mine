# Lab 화면 방향 정리

> `The Lab`의 역할 문서와 분리해, Sprint 3 기준 실험실 화면의 감정과 상호작용을 따로 정리한 메모.
> 이 문서는 `금고에서 꺼낸 원석 1개를 프로젝트 개요로 발전시키는 화면`이라는 방향을 고정하고, 이후 실제 UI 설계와 구현의 기준점으로 사용한다.

> [!summary]
> Lab은 복잡한 분석실이 아니라 `원석 하나를 붙잡고 프로젝트 개요로 다듬는 민트 블루 작업대`로 표현한다.
> 별도 스캔 단계는 두지 않고, 기존 원석 설명과 키워드를 재사용해 바로 `개요서 만들기`로 이어지게 한다.

## 왜 별도 스캔 단계를 두지 않는가

- 처음에는 `스캔 -> 분석 -> 개요서 만들기` 흐름도 고려했다.
- 하지만 실험실의 핵심 가치는 `분석을 오래 보여주는 것`보다 `아이디어를 문서로 발전시키는 것`에 있다.
- 사용자가 실험실에 들어온 뒤 또 한 번 스캔 버튼을 눌러야 하면, 초반 UX가 늘어지고 핵심 행동이 흐려질 수 있다.
- 따라서 화면은 `분석 중인 연구실처럼 보이게` 만들되, 실제 행동 흐름은 `바로 개요서 만들기`가 되도록 설계한다.

## 핵심 컨셉

- `민트 블루 장난감 연구실`
- `원석 하나에 깊게 집중하는 작업대`
- `귀여운 장치들은 가장자리`, `핵심 행동은 중앙`

Vault가 `내가 모아둔 보물함`이라면, Lab은 `그중 하나를 붙잡고 다듬는 작업대`처럼 느껴져야 한다.

## 화면의 1차 목표

- 사용자가 `이제 이 원석으로 개요서를 만든다`는 흐름을 즉시 이해하게 한다
- 별도 분석 단계 없이 바로 개요 생성으로 넘어가게 한다
- 20~40초의 생성 대기 시간을 `지루한 대기`가 아니라 `문서를 다듬는 작업 과정`처럼 느끼게 한다

## 첫 화면 구조

### 상단 카피

- 메인 문구: `이 원석을 프로젝트 개요로 다듬어볼까요?`

이 문구는 금고에서 가져온 원석을 `실험실에서 다음 단계로 발전시킨다`는 역할을 가장 자연스럽게 설명한다.

### 중앙 작업대

- 민트 블루 작업 패드 위에 원석 1개를 크게 배치
- 사용자가 지금 다루고 있는 대상은 항상 하나뿐이라는 점을 명확히 보여준다
- 화면의 시선 중심은 언제나 원석과 CTA에 머물도록 유지한다

### 원석 정보

- 원석 아래에는 `기존 원석 설명 1~3줄`을 그대로 재사용
- 그 아래에는 `키워드 칩`을 배치
- 추가 분석 문장이나 새 API 호출 결과는 여기서 만들지 않는다

핵심은 `이미 있는 정보로 충분히 이해 가능하게 만들고`, 새로 뭔가를 읽게 하기보다 바로 개요 생성으로 밀어주는 것이다.

### 메인 CTA

- 버튼 문구: `이 원석으로 개요서 만들기`

버튼은 실험실에서 가장 크고 선명한 액션이어야 한다.
보조 액션이 필요하더라도 시각적 우선순위는 낮게 유지한다.

## 왜 기존 원석 설명을 재사용하는가

- 원석 단계에서 이미 아이디어의 핵심 설명 1~3줄이 제공된다
- 이 내용을 다시 요약하거나 재해석하기 위해 별도 API를 호출하는 것은 비용과 대기 시간 측면에서 비효율적이다
- 실험실 첫 화면은 `새 해석`보다 `문서화 진입`이 목적이므로, 기존 설명과 키워드만으로 충분하다

즉, 실험실의 첫 화면은 `원석 상세 + 다음 행동 CTA`에 가깝다.

## 비주얼 톤

- 배경은 Vault보다 밝고 선명한 `민트 + 스카이 블루`
- 중앙은 깔끔한 작업대
- 귀여운 장치는 가장자리에만 배치

### 장치 배치 방향

- 왼쪽: 유리관, 버블, 작은 램프
- 오른쪽: 미니 모니터, 둥근 버튼, 작은 패널
- 전체는 `아날로그 장난감 연구실 + 가벼운 디지털 장치`의 혼합형

중요한 점은 소품이 분위기를 만들되, 중앙의 원석과 CTA를 방해하지 않아야 한다는 것이다.

## 생성 중 화면 방향

개요서 생성 API는 약 20~40초의 대기 시간이 발생할 수 있으므로, 이 구간을 설계적으로 보완해야 한다.

### 선택한 방식

- `혼합형 로딩 연출`

### 생성 시작 직후

- 원석이 작업대 위에서 짧게 반짝인다
- 램프가 점등되고, 유리관에 작은 거품이 오른다
- 버튼이나 패널이 짧게 반응한다

처음 3~5초는 `실험실이 작동하기 시작했다`는 감정이 전달되면 충분하다.

### 본 대기 상태

- 아래에서 `개요서 틀`이 올라와 자리 잡는다
- 실제 결과 텍스트를 미리 쓰지 않고, 문서의 빈 구조만 먼저 보여준다
- 사용자는 `지금 문서를 만드는 중`이라는 감각을 받게 된다

예시 섹션:

- 문제 정의
- 타깃 사용자
- 핵심 기능
- 수익 구조

### 대기 중 문구

- `원석의 결을 정리하는 중이에요`
- `프로젝트 개요 틀을 맞추고 있어요`
- `읽기 좋은 문서로 다듬는 중이에요`

이 문구들은 `실제로 무슨 작업을 하는지`를 설명해야 하며, 가짜 진행률처럼 보이지 않게 운영한다.

## 완료 전환 방식

생성 완료 후에는 결과를 바로 문서에 집중시켜야 하므로, 별도의 추가 탭 없이 자동 전환한다.

### 선택한 방식

- `짧은 완료 카드 연출 후, 개요서 상세 화면으로 자동 이동`

### 완료 순간

- 아래에서 개요서 카드가 짧게 올라온다
- 완료 문구를 보여준다: `원석이 빛나는 프로젝트 개요로 정리됐어요`
- 짧은 지연 뒤 개요서 전체 화면으로 넘어간다

이 구조는 `완성의 보상감`은 살리면서, 사용자가 결과를 보기 위해 한 번 더 탭해야 하는 번거로움은 줄인다.

## 화면별 와이어프레임

### 1. 진입 화면

```text
┌─────────────────────────────────────┐
│ The Lab                             │
│ 이 원석을 프로젝트 개요로            │
│ 다듬어볼까요?                       │
│                                     │
│      [램프]         [미니 모니터]    │
│                                     │
│         ┌──────────────┐            │
│         │  민트 작업대  │            │
│         │    원석 1개   │            │
│         └──────────────┘            │
│                                     │
│  아이디어 설명 1~3줄                 │
│  기존 원석 설명을 여기서 재사용      │
│                                     │
│  [AI] [생산성] [모바일] [구독]       │
│                                     │
│ [   이 원석으로 개요서 만들기    ]   │
└─────────────────────────────────────┘
```

### 2. 생성 중 화면

```text
┌─────────────────────────────────────┐
│ The Lab                             │
│ 프로젝트 개요로 다듬는 중이에요      │
│                                     │
│    [유리관]      [원석]    [패널]    │
│      거품         반짝임     점등     │
│                                     │
│  원석의 결을 정리하는 중이에요       │
│                                     │
│  ┌───────────────────────────────┐  │
│  │ 프로젝트 개요서                │  │
│  │ - 문제 정의                    │  │
│  │ - 타깃 사용자                  │  │
│  │ - 핵심 기능                    │  │
│  │ - 수익 구조                    │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

### 3. 완료 직전 연출

```text
┌─────────────────────────────────────┐
│ 원석이 빛나는 프로젝트 개요로        │
│ 정리됐어요                          │
│                                     │
│      [완성된 개요서 카드가 올라옴]    │
│                                     │
│        -> 개요서 화면으로 이동       │
└─────────────────────────────────────┘
```

- `이 원석을 프로젝트 개요로 다듬어볼까요?`
- `이 원석으로 개요서 만들기`

### 생성 중 화면

- `프로젝트 개요로 다듬는 중이에요`
- `원석의 결을 정리하는 중이에요`
- `프로젝트 개요 틀을 맞추고 있어요`
- `읽기 좋은 문서로 다듬는 중이에요`

### 완료 문구

- `원석이 빛나는 프로젝트 개요로 정리됐어요`

## 구현 시 주의점

- 생성 중에는 실제 API 결과가 도착하기 전까지 완성된 문장이나 가짜 요약을 미리 보여주지 않는다
- 로딩 애니메이션은 과하지 않게, 20초 이상 보더라도 피로하지 않은 수준으로 유지한다
- 중앙 작업대와 CTA가 항상 핵심이 되도록, 가장자리 소품의 시각적 존재감을 제한한다
- 개요서 생성은 실험실의 핵심 가치이므로, 진입부터 CTA까지의 흐름을 최대한 짧게 유지한다
