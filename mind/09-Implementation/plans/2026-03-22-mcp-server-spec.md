# IDEA MINE MCP Server — 설계 스펙

> 영구 무료 획득 채널로서의 MCP 서버. 앱의 핵심 루프를 텍스트로 체험시키고, 풀 경험은 앱으로 유도.

---

## 1. 포지셔닝

### 역할

MCP 서버는 **제품이 아니라 획득 채널**이다.

- 앱의 핵심 가치("키워드 조합 → 아이디어 생성")를 마찰 없이 체험
- 결과는 휘발성 — 저장/축적은 앱에서만 가능
- MCP에서 가치를 느낀 사용자가 자연스럽게 앱으로 이동

### 타겟

1차: AI 코딩 도구 사용자 (Claude Code, Cursor, Windsurf)
2차: AI 채팅 사용자 (Claude Desktop 등) — 1차 안정화 후 확장

---

## 2. 아키텍처

```
[AI 어시스턴트]
  Claude Code / Cursor / Claude Desktop
         │
[IDEA MINE MCP 서버]
  Python (FastMCP)
  역할: API 호출 + MCP 포맷 변환 + 제한 안내
  비즈니스 로직: 0줄
         │
[IDEA MINE 백엔드 API]
  Python
  역할: 모든 비즈니스 로직
         │
    ┌────┼────┐
 OpenAI  Supabase  Rate Limiter
```

### 핵심 원칙

1. **MCP에 비즈니스 로직 0줄.** 광맥 생성, 아이디어 생성, 개요서 생성은 전부 백엔드 API.
2. **앱과 MCP가 같은 API 호출.** 프롬프트 수정, 키워드 추가, 4군 규칙 변경 → 백엔드만 고치면 양쪽 반영.
3. **로그인 불필요.** 일일 제한은 IP 기반 소프트 리밋.

## 3. 백엔드 API 엔드포인트

MCP가 호출하는 API. 앱도 동일한 엔드포인트 사용.

```
GET  /api/veins/today        → 오늘의 광맥 3개 반환
POST /api/veins/reroll       → 광맥 리롤 (남은 횟수 반환)
POST /api/mine               → 채굴 (원석 10개 생성)
  body: { vein_id: string }
POST /api/overview           → 프로젝트 개요서 생성
  body: { gem_id: string }
```

### 응답 구조 (예시)

**GET /api/veins/today:**
```json
{
  "veins": [
    {
      "id": "vein_001",
      "name": "감정 케어 광맥",
      "keywords": [
        { "category": "Who", "label_ko": "1인 가구", "label_en": "Single-person Household" },
        { "category": "Domain", "label_ko": "멘탈헬스", "label_en": "Mental Health" },
        { "category": "Tech", "label_ko": "모바일 앱", "label_en": "Mobile App" },
        { "category": "Value", "label_ko": "감정 케어", "label_en": "Emotional Care" },
        { "category": "Money", "label_ko": "구독형 (SaaS)", "label_en": "Subscription (SaaS)" }
      ],
      "rarity": "normal"
    }
  ],
  "rerolls_remaining": 2
}
```

**POST /api/mine:**
```json
{
  "gems": [
    {
      "id": "gem_001",
      "title": "AI 감정 일기 앱",
      "summary": "1인 가구를 위한 음성 기반 감정 기록 서비스...",
      "keywords_used": ["1인 가구", "멘탈헬스", "모바일 앱"],
      "group": "stable"
    }
  ],
  "mining_remaining": 0
}
```

## 4. MCP 도구 설계

### 도구 4개

#### show_veins

```
이름: show_veins
설명: "첫 번째로 실행하세요. 오늘의 광맥(키워드 조합) 3개를 보여줍니다."
파라미터: 없음
호출: GET /api/veins/today
반환: 광맥 3개 (키워드 목록 + 희귀도)
```

#### reroll

```
이름: reroll
설명: "광맥이 마음에 안 들면 새로 뽑습니다. 하루 2회 가능."
파라미터: 없음
호출: POST /api/veins/reroll
반환: 새 광맥 3개 + 남은 리롤 횟수
제한 도달 시: "오늘 리롤을 모두 사용했어요. 내일 새 광맥이 열립니다.
              앱에서는 Lite/Pro로 더 많은 리롤이 가능해요. ideamineai.com"
```

#### mine

```
이름: mine
설명: "마음에 드는 광맥을 선택하면 원석(아이디어) 10개를 채굴합니다. 하루 1회."
파라미터: vein_id (string, 필수)
호출: POST /api/mine { vein_id }
반환: 원석 10개 (제목 + 요약 + 사용된 키워드 태그)
제한 도달 시: "오늘 채굴을 이미 완료했어요. 내일 새 광맥에서 다시 채굴할 수 있어요.
              앱에서는 Lite로 하루 5회, Pro로 무제한 채굴이 가능해요. ideamineai.com"
```

#### overview

```
이름: overview
설명: "원석(아이디어) 하나를 골라 프로젝트 개요서를 만듭니다. 하루 1회."
파라미터: gem_id (string, 필수)
호출: POST /api/overview { gem_id }
반환: 프로젝트 개요서 (문제 정의, 타겟, 핵심 기능, BM 등)
응답 후 안내: "이 아이디어를 금고(저장소)에 보관하려면 ideamineai.com에서 가능합니다.
              앱에서는 감정(시장성 평가), 실행 설계, MVP 청사진까지 만들 수 있어요."
제한 도달 시: "오늘 개요서를 이미 생성했어요.
              앱에서는 Lite로 여러 번, Pro로 무제한 생성이 가능해요. ideamineai.com"
```

## 5. 세계관 톤: 괄호 병기

MCP에서는 세계관 용어를 쓰되, 괄호로 일반 용어를 병기.

| 세계관 용어 | MCP에서 표시 |
|-----------|------------|
| 광맥 | 광맥(키워드 조합) |
| 원석 | 원석(아이디어) |
| 채굴 | 채굴(생성) |
| 금고 | 금고(저장소) |
| 감정 | 감정(시장성 평가) |
| 리롤 | 리롤(새로 뽑기) |

이유:
- 세계관 없이 "아이디어 생성기"라고만 하면 IDEA MINE의 정체성이 사라짐
- 괄호 병기로 처음 쓰는 사용자도 즉시 이해
- 앱으로 넘어갔을 때 "아, 광맥이 이거구나" 연결이 자연스러움

## 6. 온보딩 흐름

사용자가 도구 이름을 몰라도 됨. AI 어시스턴트가 도구 설명을 읽고 자연스럽게 안내.

### MCP 서버 설명 (AI가 읽는 메타데이터)

```
"IDEA MINE - 6가지 키워드를 조합해 AI 사업 아이디어를 채굴하는 도구.
 사용 흐름: show_veins로 광맥 확인 → mine으로 채굴 → overview로 개요서 생성.
 마음에 안 드는 광맥은 reroll로 새로 뽑을 수 있습니다."
```

### 실제 대화 예시

```
사용자: "사업 아이디어 좀 만들어줘"

AI: "IDEA MINE으로 아이디어를 채굴해볼게요. 먼저 오늘의 광맥을 확인합니다."
    → show_veins() 호출

AI: "오늘의 광맥(키워드 조합) 3개입니다:

     1번 광맥 — 감정 케어 광맥
     [1인 가구 · 멘탈헬스 · 모바일 앱 · 감정 케어 · 구독형]

     2번 광맥 — 실버 헬스 광맥
     [시니어 · 헬스케어 · 웨어러블 · 건강 트래킹 · B2B SaaS]

     3번 광맥 — 크리에이터 자동화 광맥
     [1인 크리에이터 · 마테크 · 자동화 워크플로우 · 시간 단축 · 구독형]

     어떤 광맥이 끌리세요? 마음에 안 들면 리롤(새로 뽑기)도 가능해요."

사용자: "1번으로 채굴해줘"

AI: → mine(vein_id="vein_001") 호출

AI: "1번 광맥에서 원석(아이디어) 10개를 채굴했습니다:

     1. AI 감정 일기 앱
        [1인 가구 · 멘탈헬스 · 모바일 앱]
        하루 한 번 음성으로 감정을 기록하면...

     2. 혼밥 감정 케어 서비스
        [1인 가구 · 감정 케어 · 구독형]
        혼자 식사하는 시간을 감정 체크인 루틴으로...

     ...

     개요서를 만들어볼 원석이 있나요?"

사용자: "1번으로 개요서 만들어줘"

AI: → overview(gem_id="gem_001") 호출

AI: "프로젝트 개요서가 완성됐어요:

     [문제 정의] 1인 가구의 감정 관리 부재...
     [타겟] 20-30대 1인 가구...
     [핵심 기능] ...
     [BM] 구독형...

     ---
     이 아이디어를 금고(저장소)에 보관하려면 ideamineai.com에서 가능합니다.
     앱에서는 감정(시장성 평가), 실행 설계, MVP 청사진까지 만들 수 있어요."
```

## 7. 기능 범위 + 앱 유도

### MCP에서 가능 (= 앱 Free와 동일)

- 오늘의 광맥 3개
- 리롤 2회
- 채굴 1회 → 원석 10개 (4군, 키워드 3~5개 사용)
- 개요서 1회
- AI 키워드 잠금 (5원석만)

### 앱 유도 메시지 (도구 응답에 포함)

| 시점 | 메시지 |
|------|--------|
| 채굴 후 | "이 원석들을 금고에 보관하려면 ideamineai.com에서" |
| 개요서 후 | "감정, 실행 설계, MVP 청사진은 앱에서 가능해요" |
| 채굴 제한 | "앱에서 Lite(5회/일) 또는 Pro(무제한)로 더 채굴" |
| 개요서 제한 | "앱에서 Lite로 여러 번, Pro로 무제한 생성" |
| 리롤 제한 | "내일 새 광맥이 열려요. 앱에서는 더 많은 리롤 가능" |

### 전환 핵심 동기

**결과가 휘발성** — MCP에서는 저장 불가. "좋은 아이디어인데 내일 다시 못 보겠네" → 앱 설치.
횟수 제한은 "짜증", 휘발성은 "아쉬움". 아쉬움이 더 강한 전환 동기 (Loss Aversion).

## 8. Rate Limiting

| 제한 | 수치 | 방식 | 이유 |
|------|------|------|------|
| 채굴 | 1회/일 | IP 기반 | 앱 Free와 동일 |
| 개요서 | 1회/일 | IP 기반 | 앱 Free와 동일 |
| 리롤 | 2회/일 | IP 기반 | 앱 Free와 동일 |
| 전체 호출 | 10회/시간 | IP 기반 | 악용 방지 |

- 로그인/계정 불필요
- IP 기반 소프트 리밋 (우회 가능하지만, 무료 채널이므로 엄격할 필요 없음)
- 일일 리셋: UTC 00:00

## 10. 향후 확장 (Phase 2+)

### 계정 연동

앱 계정이 있는 유저가 MCP에서 인증하면:
- MCP 채굴 결과가 앱 금고에 자동 저장
- Lite/Pro 혜택이 MCP에서도 적용 (추가 채굴, AI 키워드 등)
- MCP가 "앱의 CLI 버전"으로 진화

### 추가 도구 후보

- `appraise(gem_id)` — 기본 감정 (Lite/Pro 연동 시)
- `my_veins()` — 최근 채굴 히스토리 (계정 연동 시)
- `custom_mine(keywords)` — 커스텀 키워드 채굴 (Lite/Pro 연동 시)

## 11. 기술 스택

| 컴포넌트 | 기술 | 이유 |
|----------|------|------|
| MCP 서버 | Python + FastMCP | 백엔드와 같은 언어. MCP 래퍼가 얇으므로 빠르게 개발 |
| 백엔드 API | Python (FastAPI 또는 Flask) | 기존 스택. 앱과 MCP 공용 |
| AI | OpenAI API | 기존 선택 |
| DB | Supabase | 기존 선택. 광맥/세션 상태 저장 |
| Rate Limit | IP 기반 인메모리 또는 Supabase | 단순 구현 |
