# The Lab (실험실)

> Mine에서 고른 아이디어 원석을 문서 자산으로 정제하는 공간. v2에서 Lab의 역할은 "직접 조합"이 아니라 "선택한 신호를 이해하고 평가하고 확장"하는 것이다.

---

## 공간 목표

- 선택한 아이디어를 빠르게 이해 가능한 문서로 바꾼다
- 여러 관점의 평가를 통해 다음 행동을 돕는다
- 나중에 바로 구현 가능한 수준의 깊은 문서까지 확장한다
- Free/Lite/Pro 차이를 **문서 깊이와 판단 품질**로 체감시킨다

---

## v2에서 Lab이 하는 일

Lab은 더 이상 키워드를 직접 고르고 아이디어를 생성하는 공간이 아니다.

- **Mine:** 광맥을 고르고 아이디어 원석을 채굴
- **Vault:** 원석을 저장하고 다시 꺼내봄
- **Lab:** 선택한 원석을 문서 자산으로 정제

즉, Lab의 입력은 `선택된 아이디어`이고, 출력은 `Overview`, `Appraisal`, `Full Overview`다.

---

## 현재 Phase 1 파이프라인

### 1단계. 프로젝트 개요 (Overview)

**역할:** "이 아이디어가 어떤 프로젝트가 될 수 있는지 빠르게 이해하는 단계"

- 입력: Mine 또는 Vault에서 고른 아이디어 1개
- 출력: 문제 정의, 타겟, 핵심 기능, 차별점, 수익 구조, MVP 범위
- 성격: 짧고 선명한 기획 초안
- 목표: 1~2분 안에 전체 그림을 파악하게 만들기

### 2단계. 감정 (Appraisal)

**역할:** "이 프로젝트를 여러 관점으로 읽어주는 단계"

- 입력: 생성된 Overview
- 출력: 시장성, 문제 적합성, 실행 가능성, 차별화, 확장성, 리스크 관련 코멘트
- 성격: 점수표보다 짧고 날카로운 판단 카드
- 목표: 강점과 약점을 빠르게 입체적으로 보이기

### 3단계. 풀 개요서 (Full Overview) — Pipeline v2

**역할:** "이 문서 하나만 있으면 바이브 코딩으로 MVP를 뚝딱 만들 수 있는 입력물"

- 입력: Overview + Appraisal을 거친 아이디어
- 출력: Narrative 블록(9섹션) + Technical 블록(6섹션)이 합쳐진 실행 문서
- 성격: Claude Code, Cursor 등 AI 코딩 도구에 복사하면 **바로 코딩이 시작되는** 수준의 문서
- 목표: 아이디어 → MVP까지의 거리를 최소화. "좋은 아이디어"를 "바로 만들 수 있는 설계서"로 전환
- 핵심 품질 기준: **"이 문서를 AI 코딩 도구에 넣었을 때 바로 프로젝트가 시작되는가?"**

**v2 파이프라인** (0to1log 핸드북 패턴 적용):

```
축 분류 (gpt-5-nano) → 섹션 가중치 생성 (Python)
  → 풀 개요 생성 (gpt-5, depth guide 포함)
  → Self-Critique (gpt-5-mini, 점수 < 70이면 재생성)
  → DB 저장
```

**3축 분류:** 아이디어의 Interface/Business/Technical 복잡도를 high/medium/low로 측정.
→ 축 값에 따라 15섹션 각각의 깊이가 달라짐.
→ "감정 일기 앱"과 "기업용 SaaS"가 같은 템플릿이지만 다른 깊이로 생성.

상세: `2026-04-06-full-overview-pipeline-v2`

## 티어별 깊이

### Free — "이런 게 가능하구나"

- 개요 3/일 + 기본 감정 3축
- 풀 개요/MVP 청사진은 **미리보기 UI**로만 노출 (블러 + 업그레이드 CTA)
- Lab에 들어오면 "더 깊은 문서가 존재한다"는 걸 인지하게 함

### Lite ($4.99) — "이 아이디어의 전체 그림이 보인다"

- 개요 10/일 + 감정 6축 + **풀 개요 3/월**
- 풀 개요로 "이 제품이 뭔지" 완전히 이해 가능
- MVP 청사진은 **미리보기 UI**로 노출 → Pro 전환 유도
- "이것만으로도 충분히 유용한" 플랜. 부족한 요금제로 느끼면 안 됨

### Pro ($12.99) — "이 문서로 바로 만들 수 있다"

- 개요 30/일 + 정밀 감정 6축 + 풀 개요 무제한 + **MVP 청사진 + Phase 로드맵**
- MVP 청사진 = 바이브 코딩의 최종 입력물. Claude Code에 넣으면 MVP가 나옴
- Pro의 핵심 가치 = "아이디어 → 만들 수 있는 설계서"까지의 완전한 파이프라인

## Lab의 장기 확장

Phase 1의 Lab은 `Overview -> Appraisal -> Full Overview`까지를 핵심으로 잡는다.

그 이후에는 아래 단계를 **Lab의 확장 파이프라인**으로 추가한다.

### 4단계. MVP 청사진 (Pro only) — Phase 2 Sprint 6

**역할:** "풀 개요서를 AI 코딩 도구에 넣으면 바로 MVP가 나오는 실행 설계서"

- 입력: 풀 개요서
- 출력: 기술 스택 확정 + Must/Should/Later 구체화 + Sprint 단위 구현 순서 + 검증 포인트
- 성격: 바이브 코딩의 최종 입력물. 이 문서가 Pro의 핵심 과금 요소
- 품질 기준: "이 문서를 Claude Code에 넣고 '이거 만들어줘'라고 하면 진짜 만들어지는가?"

풀 개요서와의 차이:
- 풀 개요서 = **"이 제품이 뭔지"** (What)
- MVP 청사진 = **"어떤 순서로 만들지"** (How + When)

### 5단계. 방향 정리 (미래 확장)

- **코치형 정리:** 다음에 무엇을 더 좁히고 강조해야 하는지

이 확장 단계들은 현재 Phase 1의 기본 체인을 대체하는 것이 아니라, 이미 정제된 자산 위에 더 깊이를 쌓는 방향으로 붙는다.

---

## 설계 원칙

- Lab은 "아이디어를 뽑는 공간"이 아니라 "선택한 아이디어를 자산화하는 공간"이다
- 문서는 길어지는 것보다 **다음 행동을 더 쉽게 만드는 것**이 우선이다
- 미래 확장 단계는 지금의 3단계 체인 위에 자연스럽게 올라가야 한다
- 세계관은 실험실이지만, 사용감은 분석 도구에 가까워야 한다
