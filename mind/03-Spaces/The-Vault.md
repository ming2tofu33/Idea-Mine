# The Vault (금고)

> 결과를 저장하고 다시 보는 공간. "이 서비스가 남는 툴"이라는 감각을 만든다.

---

## 공간 목표

- 아이디어와 개요서가 안전하게 보관된다는 신뢰감
- 내가 쌓아둔 성과가 눈에 보이는 축적감
- 과거 채굴 기록을 다시 찾고 활용하는 재탐색

---

## 핵심 기능

### 저장한 아이디어 목록

- 원석(아이디어 draft) 목록 관리
- 상태 배지: draft, 개요서 완성, 감정 완료

### 상세 개요서 보관

- 프로젝트 개요 문서 열람/관리
- 실험실에서 생성된 문서 자동 연결

### 운반 시스템 (가방 / 광차)

채굴한 원석 10개 중 Vault로 가져갈 수 있는 수를 결정하는 운반 도구.

| 티어 | 운반 도구 | 채굴당 운반 용량 | 설명 |
|------|----------|----------------|------|
| Free | **가방 (Bag)** | 2개 (레벨업 → 최대 5개) | 광부가 주머니에 챙겨가는 느낌 |
| Lite | **광차 (Mine Cart)** | 10개 전부 | 레일 위 광차에 실어가는 느낌 |
| Pro | **광차 (Mine Cart)** | 10개 전부 | 레일 위 광차에 실어가는 느낌 |

- Free: 10개 중 골라야 하는 긴장감 → 광차 업셀 포인트
- Lite/Pro: 전부 가져갈 수 있지만 Vault 저장 용량이 제한

### Vault 저장 용량

| 티어 | 저장 용량 |
|------|----------|
| Free | 제한적 (소용량) |
| Lite | 넉넉 |
| Pro | 대용량 |

- 운반은 광차로 자유로워도 Vault가 꽉 차면 정리 필요
- Vault 확장은 우선 Lite/Pro 구독 경계로 운영하고, 향후 프리미엄 경제 레이어와 연동 가능

### 과거 채굴 기록

- 지금까지의 채굴 히스토리
- 즐겨찾기 조합

### 금고 확장 슬롯

- 저장 공간 확장 (MVP는 Lite/Pro 중심, 이후 프리미엄 경제 레이어 연동 가능)

## Vault = 저장소가 아니라 허브

Vault는 끝나는 곳이 아니라 **다음 단계로 보내는 허브**다.

### 문서 단계

| 단계 | 이름 | 상태 |
|------|------|------|
| 1 | **원석** (Raw Gem) | 아직 다듬기 전 |
| 2 | **개요 문서** (Overview) | 감정하고 정리된 상태 |
| 3 | **확장 문서** (Blueprint) | 실행 준비 단계 |

### 핵심 액션

| 액션 | 설명 | Phase |
|------|------|-------|
| **실험실로 보내기** | 원석 → 개요 문서로 발전. Vault의 가장 중요한 버튼 | Phase 1 |
| **비교하기** | 원석 2~3개 또는 개요 문서 나란히 비교. 시장성/선명도 비교 | Phase 3 (Pro) |
| **분기 만들기** | 같은 원석으로 BM/타겟만 바꿔서 새 방향 발전 | Phase 3 (Pro) |

### 설계 원칙

- 기획자는 비교하면서 고르니까, 비교 기능이 있으면 실무성이 살아남
- 분기는 저장한 걸 복사해서 새 방향으로 발전시키는 것 — IDEA MINE다운 기능
- 비교/분기는 유료(Pro) 기능으로 적합

---

## MVP 금고 구조

MVP에서는 **저장 + 실험실 연결**에 집중.
- 비교/분기는 Phase 3 (Pro)
- 폴더/컬렉션은 Phase 2 이후
