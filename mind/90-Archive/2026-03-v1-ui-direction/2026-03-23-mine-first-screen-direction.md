# 광산 첫 화면 방향 정리

> 광산 첫 화면을 `설명형 대시보드`보다 `게임처럼 보이는 고정 히어로 씬`으로 재정의한 메모.
> 이 문서는 기존 `2026-03-22-mine-home-screen.md`의 카드 중심 초안을 보정하고, Phase 1 구현의 기준점으로 사용한다.

> [!summary]
> Phase 1의 광산 첫 화면은 `사선 픽셀 광산 내부 씬 + 벽에 박힌 오늘의 광맥 3개 + 하단 CTA` 구조로 간다.
> 캐릭터 이동은 아직 넣지 않고, 사용자는 광맥 오브젝트를 직접 탭해 선택한 뒤 하단 버튼으로 채굴을 시작한다.

## 왜 이 방향이 맞는가

- 제품의 핵심은 `광맥 선택 -> 아이디어 생성 -> 저장` 흐름이지, 실시간 이동 플레이가 아니다.
- 하지만 첫 화면은 단순한 카드 나열보다 `광산에 들어와 있다`는 감각을 강하게 주는 편이 브랜드에 더 잘 맞는다.
- 따라서 Phase 1에서는 `게임 플레이`보다 `게임처럼 보이는 앱`을 목표로 한다.
- 픽셀 세계관은 강하게 유지하되, 조작은 명확한 탭/버튼 기반으로 단순화한다.

## Phase 1 원칙

- 광부 캐릭터 이동 없음
- 탐험형 맵 없음
- 오브젝트 직접 탭은 허용
- 결과적으로는 `픽셀 광산 허브 씬을 가진 탭 기반 앱`

## 시각 방향

- 시점은 완전 평면 정면보다 `약간 사선인 픽셀 광산 씬`
- 완전 자유 이동형 아이소메트릭 게임으로 확장하지는 않음
- 느낌은 `깊은 내부 광산`, `랜턴 불빛`, `레일과 광차`, `벽면 광맥`
- 목표는 `광산 깊숙한 곳에서 오늘의 광맥을 고르는 장면`

## 첫 화면의 핵심 장면

- 공간 타입: `광산 깊은 내부 공간형`
- 화면 중심: `랜턴이 있는 작업 교차로`
- 구조 요소: `레일이 만나는 바닥`, `작은 광차 1~2개`, `작업용 목재 구조물`
- 캐릭터: 광부 캐릭터가 항상 화면 안에 있음
- 캐릭터 상태: 일단은 `가만히 서 있는 idle`

중앙 허브는 움직이는 게임 공간이 아니라, `오늘의 광맥을 고르는 상징적 무대`로 본다.

## 오늘의 광맥 3개

- 오늘의 광맥 3개는 항상 화면에 보인다
- 숨겨진 메뉴처럼 펼쳐지지 않는다
- 광맥은 카드가 아니라 `벽에 박힌 광맥 오브젝트`로 표현한다
- 광맥 3개는 삼각형 배치로 둔다

### 배치 원칙

- 상단 중앙: 광맥 1개
- 중하단 좌측 벽: 광맥 1개
- 중하단 우측 벽: 광맥 1개
- 맨 아래는 CTA 안전 영역으로 비워둔다

즉, 개념은 `좌하 / 우하 / 상단 중앙`이지만 실제 배치는 `상단 중앙 / 중하단 좌 / 중하단 우 / 최하단 CTA` 구조로 정리한다.

## 광맥 라벨 방식

- 각 광맥에는 항상 캡션 박스가 붙는다
- 캡션은 광맥 가까이에 두되, 전체 레이아웃은 화면 안쪽으로 정돈한다
- 초반에는 라벨을 숨기지 않는다

### 캡션 박스 구성

- 광맥 이름: 세계관형 이름
- 짧은 설명: 직관적인 한 줄 설명
- 희귀도 태그: 색상 + 작은 아이콘

### 예시

- `반짝 반응 광맥`
- `반응이 빠르게 오는 아이디어가 나올 것 같아요`
- `[✦ 반짝]`

## 희귀도 표현

희귀도는 텍스트만으로 구분하지 않고 `색상 + 작은 아이콘`을 함께 사용한다.

- 일반: 차분한 회청 계열 + 작은 돌/광석 아이콘
- 반짝: 골드 계열 + 반짝이 아이콘
- 희귀: 보라/핑크-퍼플 계열 + 작은 보석 아이콘

## 인터랙션 흐름

1. 사용자는 벽에 박힌 광맥 오브젝트 중 하나를 탭한다
2. 선택된 광맥이 강조된다
3. 선택되지 않은 나머지 광맥 2개는 살짝 어두워진다
4. 하단 CTA가 활성화된다
5. 사용자가 CTA를 눌러 채굴을 시작한다

## 선택 상태 표현

- 선택된 광맥 자체가 밝아진다
- 선택된 광맥이 박힌 `벽면 전체가 은은하게 살아나는` 느낌을 준다
- 나머지 2개는 살짝 어둡게 눌러 집중도를 올린다
- 캡션 박스와 희귀도 태그도 선택 상태에 맞춰 선명도 차이를 준다

## CTA 규칙

- 선택 전: `광맥을 선택하세요`
- 선택 후: `"{광맥 이름} 채굴 시작"`
- 버튼은 맨 아래 고정 도크에 위치
- 오터치를 막기 위해 `광맥 선택 -> CTA 한 번 더 누르기` 구조를 유지

이 방식으로 `세계관 몰입감`과 `앱형 명확함`을 동시에 확보한다.

## 화면 구조 요약

```text
┌──────────────────────────────────┐
│          The Mine                │
│        깊은 내부 광산 씬          │
│                                  │
│           [상단 중앙 광맥]         │
│          [이름/설명/희귀도]        │
│                                  │
│   [좌측 벽 광맥]   광부+랜턴   [우측 벽 광맥]
│   [이름/설명/태그] 레일+광차   [이름/설명/태그]
│                                  │
│                                  │
│      [{선택된 광맥 이름} 채굴 시작] │
└──────────────────────────────────┘
```

## UX 원칙

- 첫 화면에 들어오면 사용자는 `오늘의 선택지 3개`를 즉시 이해할 수 있어야 한다
- 세계관 용어는 유지하되, 설명은 직관적으로 써서 초기 학습 비용을 낮춘다
- 오브젝트를 직접 누르는 감각은 살리되, 실제 터치 영역은 넉넉하게 잡는다
- 화면은 게임처럼 보여야 하지만, 사용 흐름은 헷갈리지 않아야 한다

## Phase 1에서 하지 않는 것

- 캐릭터 직접 이동
- 맵 탐색
- 광산 입구 진입 후 광맥 펼치기
- 라벨 숨김 UI
- 선택 즉시 바로 채굴
- 모든 오브젝트를 상호작용 가능한 메뉴로 만들기

## 이후 확장 가능성

### Phase 2 이후 후보

- 광부의 아주 작은 idle 애니메이션
- 광맥 선택 시 더 뚜렷한 이펙트
- 일부 오브젝트 상호작용
- 광산 내부 허브의 단계적 확장

### 먼 미래 후보

- 캐릭터 이동
- 탐험형 공간
- 광산 내부 구역 탐색

핵심은 `지금은 게임처럼 보이는 앱`, `나중에는 일부 게임 문법 확장 가능` 구조로 가는 것이다.

## 구현 메모

- `mind/03-Spaces/The-Mine.md`의 행동 중심 원칙은 유지한다
- 다만 시각 연출은 기존 카드 중심 초안보다 훨씬 공간형으로 가져간다
- `mind/08-Design/Pixel-Art-Style-Guide.md`와 `mind/08-Design/Reference-Image-Prompt-Guide.md`를 함께 참고해 에셋 방향을 맞춘다
- 광맥은 UI 카드가 아니라 `씬 안의 선택 가능한 오브젝트`로 간주한다
