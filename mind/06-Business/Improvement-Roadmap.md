# Improvement Roadmap

> `Business-Audit` 기반 개선 로드맵. 검증 -> Lean MVP -> Gate 판단 -> 확장의 Gate-Based Development.
> **상태:** 이 문서의 핵심 제안이 [[Phases-Roadmap]]에 머지됨 (2026-03-22). 실행 계획은 Phases-Roadmap을 참조.

## 현재 vs 제안

```
현재:  Phase 1A(7 Sprint) -> 1B(3) -> 2(4) -> 3(3) -> 4(무한)
제안:  Phase 0(검증) -> 1(핵심3 Sprint) -> Gate -> 나머지
```

## Phase 0 -- 검증 (2주)

코드 한 줄 쓰기 전에 해야 할 것들.

### Week 1: 만들기 + 배포

**Day 1-2: 경쟁 분석 직접 테스트**

`Competitive-Analysis` 문서 작성 완료. 추가로 직접 테스트:

ChatGPT에게 "AI 기술(음성AI), 타겟(1인가구), 산업(헬스케어), 플랫폼(앱), 가치(감정케어), 수익(구독) 조합으로 서비스 아이디어 10개 만들어줘" 요청. 결과를 보고 IDEA MINE이 제공할 차별 가치가 정말로 있는지 냉정하게 판단.

동시에: Waitlist 랜딩 제작 + 배포 (ideamineai.com). 상세 구조는 `GTM-Strategy` 참조.

**Day 3-4: 48시간 프로토타입**

만들 것: 웹 프로토타입 1페이지
- 키워드 6개 드롭다운(카테고리별 1개씩 선택)
- "채굴하기" 버튼 -> OpenAI API 호출 -> 아이디어 3-5개 표시
- 픽셀 아트 없어도 됨, 기능만 작동하면 됨

**Day 5: 트위터 BIP 첫 게시 + 테스터 모집 시작**

매일 15분. 프로토타입 스크린샷 포함한 첫 게시물로 시작.

### Week 2: 테스트 + 분석 + 가격

**Day 6-10: 유저 테스트**

주변 기획자/창업자 5-10명에게 프로토타입 링크 공유:
- 비대면: 링크 공유 후 설문 (기본)
- 대면/화상: 사용 과정을 관찰하며 반응 기록 (이상적)
- 물어볼 것: "이걸 매일 쓸 것 같아요?" / "돈 내고 쓸 의향?"
- 동시에 Van Westendorp 4질문 실시

Van Westendorp 4질문:
1. "이 기능에 월 얼마면 너무 비싸서 안 쓰겠어요?"
2. "얼마면 너무 싸서 품질이 의심돼요?"
3. "비싸지만 고려해볼 만한 가격은?"
4. "좋은 거래라고 느끼는 가격은?"

**Day 11-14: 결과 분석 + Gate 판단**

테스트 결과 정리:
- 반응이 좋으면 -> Phase 1 진행
- "괜찮은데 매일은 아니야" -> 리텐션 메커니즘 재설계 필요 (Phase 0 연장)
- "ChatGPT로 해도 되는데?" -> 차별화 강화 또는 피봇 검토

### Gate: 아래 3개 중 2개 이상 통과?

1. 테스터 5명 중 3명+ "다시 쓰고 싶다" 응답
2. Waitlist 등록 50명+ 달성
3. 테스터 5명 중 2명+ "유료라도 쓸 의향" 응답

Yes -> Phase 1. No -> 차별화 강화 후 재테스트 (Phase 0 반복).

## Phase 1 -- Lean MVP (Sprint 1-3, 약 3-4주)

### Sprint 1: DB + Auth + 디자인 토큰

현재 `Phases-Roadmap`의 Sprint 1과 동일.

### Sprint 2: 홈(광맥 3개 + 리롤) + 아이디어 생성 (5개)

- 오늘의 광맥 3개 생성 로직 (백엔드)
- 광맥 카드 UI (캐러셀) -- 3개 요소만: 상태바(미니) + 광맥3 + 리롤
- 아이디어 5개 생성 (10개에서 축소)
- 원석 선택/금고 반입

### Sprint 3: 금고 + 개요서 생성 + 간이 감정

- 금고 화면 (저장된 아이디어 목록)
- 아이디어 상세 보기
- 프로젝트 개요 생성 API
- 기본 감정 생성

### 빼는 것 (Phase 1에서 제외)

- 배지, 퀘스트, 레벨 -- 게임 경제는 핵심 루프 검증 후
- 3통화 시스템 -- 핑크 결정만 최소로
- 꾸미기, 코스튬 -- Phase 2 이후
- i18n -- 한국어만 먼저
- 사운드 이펙트 -- 선택사항
- 공사 중 배너 -- 불필요

### Gate: 첫 50명 유저의 D1 리텐션 > 20%?

Yes -> Phase 1.5. No -> 핵심 루프 개선.

## Phase 1.5 -- 리텐션 검증 (Sprint 4-5)

### Sprint 4: 데일리 광맥 + 연속 접속 + 핑크 결정(1종)

- 데일리 광맥 갱신 시스템
- 연속 접속 카운터
- 핑크 결정 기본 획득/소비
- Wordle식 공유 카드 기능 (바이럴 루프)

### Sprint 5: Pro 구독 + 웹 배포 + 전환 최적화

- RevenueCat 세팅 (모바일)
- Polar.sh 세팅 (웹)
- Free -> Pro 전환 흐름 (Loss Aversion 프레이밍 적용: 5원석 vs 6원석 결과 나란히 비교)
- Pro 7일 무료 체험 (카드 등록 불필요)
- Expo Router 웹 버전 배포 (첫 채굴 1회 무료, 로그인 불필요)
- 웹 -> 앱 전환 유도 흐름: "더 많이 채굴하려면 앱 다운로드"

### Gate: Free -> Pro 전환율 > 2%?

Yes -> Phase 2. No -> 가치 제안 재설계.

Gate 실패 시 점검 항목:
1. 5원석 vs 6원석 아이디어 품질 차이가 체감 가능한가?
2. Pro 체험 후 이탈하는 사용자의 이유가 무엇인가?
3. 가격이 너무 높은가? (Van Westendorp 재검토)

## Phase 2 -- 게임 확장 (Sprint 6-8)

이 시점에서야 게임 경제 도입. 사용자 데이터 기반으로 어떤 게임 요소가 필요한지 판단.

- 배지, 레벨 시스템
- 꾸미기 기본
- 추가 통화 (Pink Diamond)
- 프로그래매틱 SEO 페이지 자동 생성
- "아이디어 DNA 리포트" 기능

## Gate-Based Development 핵심

```
현재:  [Sprint 1] -> [Sprint 2] -> ... -> [Sprint 17] (일직선)
개선:  [Sprint 1-3] -> Gate 판단 -> [방향 결정] -> [Sprint 4-5] -> Gate -> ...
```

**각 Gate에서 묻는 질문:**
1. 사용자가 핵심 루프를 완주하는가? (채굴 -> 저장)
2. 다음 날 다시 오는가? (D1 리텐션)
3. 돈을 내려는 의향이 있는가? (Pro 전환 클릭)

Gate를 통과하지 못하면 다음 Sprint가 아니라 현재를 개선. 이게 린 방식.
