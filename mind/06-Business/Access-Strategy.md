# Access Strategy

> IDEA MINE v2의 접근 전략은 `auth-first`가 아니라 `experience-first + bounded demo + soft gate`를 기준으로 한다.
> 비로그인 사용자도 제품의 핵심 가치를 체험할 수 있어야 하지만, 반복 사용, 저장, 자산화, 심화 생성은 계정과 유료 구간에서 회수한다.

---

## 왜 이 문서가 필요한가

현재 구조에서 비로그인 사용자는 제품의 대표 공간인 Mine을 사실상 체험하지 못한다. 이 방식은 구현은 단순하지만, v2가 지향하는 온보딩, GTM, 가격 구조와 충돌한다.

IDEA MINE v2는 설명보다 먼저 체험을 주는 제품이어야 한다. 사용자는 "가입해야 뭐가 보인다"보다 "한 번 눌러보니 바로 재미와 쓸모가 느껴진다"를 먼저 경험해야 한다.

이 문서는 다음 세 가지를 동시에 맞추기 위한 기준 문서다.

1. 비로그인 사용자도 핵심 가치를 맛볼 수 있게 한다.
2. 무료와 유료의 경계를 흐리지 않도록 제한을 설계한다.
3. 공개 진입면을 콘텐츠와 획득 채널로 활용할 수 있게 한다.

---

## 핵심 결론

IDEA MINE v2의 최적 접근 전략은 다음과 같다.

- 완전 개방형 무료 제품으로 가지 않는다.
- 로그인 전 완전 차단 구조도 유지하지 않는다.
- `공개 체험면 + 제한된 첫 경험 + soft gate`의 하이브리드 구조로 간다.

이 전략의 요지는 단순하다.

> 비로그인 사용자는 제품의 핵심 가치를 체험할 수 있어야 한다.
> 하지만 저장, 반복 사용, 심화 생성, 자산화는 계정과 유료 구간에서 회수한다.

---

## 왜 `auth-first`가 맞지 않는가

### 1. 온보딩 전략과 충돌한다

[[Onboarding]]은 설명보다 첫 체험을 앞세우는 구조를 전제로 한다. 첫 광맥 확인, 첫 결과 확인, 첫 저장, 첫 개요 생성이 핵심 루프의 시작이다.

비로그인 사용자가 이 단계에 진입하지 못하면 온보딩은 사실상 랜딩 카피와 로그인 유도만 남게 된다.

### 2. Mine의 역할과 충돌한다

[[The-Mine]]은 v2의 flagship 공간이다. Mine은 "들어오면 바로 시작되는 곳"이어야 하고, 3초 안에 오늘의 광맥을 보고, 30초 안에 첫 결과를 느끼게 해야 한다.

로그인 전 진입이 완전히 막혀 있으면 Mine은 핵심 경험이 아니라 가입 이후 기능 목록 중 하나로 밀린다.

### 3. 가격 구조와 충돌한다

[[Tier-Structure]]는 Free를 "가치 체험" 구간으로 정의한다. [[Monetization-Strategy]]도 너무 이른 차단은 부자연스럽다고 본다.

비로그인 단계에서 0을 제공하고, 로그인 이후에야 1을 제공하는 구조는 `proof of value`를 만들지 못한다.

### 4. 콘텐츠 및 GTM 전략과 충돌한다

[[GTM-Strategy]]는 공개 진입면, 공유, 탐색, SEO, "오늘의 아이디어" 같은 흐름을 전제로 한다. 공개 체험면이 없으면 검색 가능한 표면과 공유 가능한 장면이 매우 약해진다.

### 5. KPI 측정 구조와도 어긋난다

[[KPI-Success-Metrics]]는 `첫 mining 완료`, `vault save rate`, `outline generation rate`를 핵심 지표로 본다. 그런데 비로그인 사용자가 첫 mining 완료조차 못 하면, 상단 퍼널에서 제품 적합성 자체를 보기 어려워진다.

---

## 왜 `완전 개방형`도 아닌가

반대로, 비로그인 사용자에게 실시간 풀 경험을 무제한 개방하는 것도 최선은 아니다.

### 1. 무료와 유료의 경계가 흐려진다

비로그인에서도 반복 탐사, 전체 결과 확인, outline 생성, 저장, 재방문이 모두 가능하면 계정 생성 이유가 약해진다.

### 2. 비용과 남용 리스크가 커진다

실시간 생성과 반복 reroll이 로그인 없이 열리면 AI 비용, 봇 남용, 비정상 트래픽 문제가 커진다.

### 3. Free의 역할이 과도하게 커진다

Free는 `가치 체험`과 `기본 반복 사용` 정도까지여야 한다. 비로그인 무료가 이미 강하면 signed-in free와 lite의 역할이 애매해진다.

### 4. 전환 포인트가 사라진다

사용자가 "지금 가입하면 무엇이 열리는지"를 분명히 느껴야 soft gate가 전환 장치로 작동한다. 완전 개방형에서는 그 긴장점이 약해진다.

## 권장 접근 모델

IDEA MINE v2는 접근 권한을 아래 네 층으로 나눈다.

### 1. Public Surface

누구나 볼 수 있는 층이다. 검색, 공유, 첫 인상, 콘텐츠 확산을 위한 표면이다.

포함:

- 랜딩
- 오늘의 광맥 또는 curated 공개 광맥 미리보기
- 공개 예시 아이디어
- 공유 가능한 idea-of-the-day / explore 페이지

목표:

- 제품이 무엇인지 빠르게 이해
- Mine의 분위기와 구조를 미리 체감
- 계정 없이도 "한 번 눌러볼 가치가 있다"는 느낌 제공

### 2. Anonymous First Experience

비로그인 사용자가 실제로 한 번 체험해보는 구간이다. 여기서는 `wow + clarity`가 목표다.

권장 범위:

- 광맥 3개 보기
- 광맥 1개 선택
- 샘플 아이디어 3개 보기
- 짧은 explanation / why this vein 보기

제한:

- 전체 10개 결과 비공개
- 저장 불가
- outline 생성 불가
- 재방문 히스토리 없음
- 반복 reroll 제한 또는 불가

### 3. Signed-in Free

비로그인보다 한 단계 더 깊은 기본 사용 구간이다. 반복 사용은 가능하지만, 빈도와 깊이는 제한한다.

포함:

- 실제 오늘의 광맥
- 제한된 mining / reroll
- 결과 저장
- 기본 Vault 사용
- 제한된 Lab 진입
- 기본 overview 또는 teaser 수준 심화

목표:

- "이 제품을 계속 쓰면 좋겠다"는 감각 형성
- 저장과 재방문을 통해 자산화 경험 시작

### 4. Paid Tiers

Lite와 Pro는 반복 사용, 깊이, 실행, 문서화, 자산화 강도를 올리는 구간이다.

포함:

- 더 많은 mining / reroll
- 더 많은 결과 / 더 깊은 outline
- 고급 appraisal / full overview / 실행 문서
- 더 넓은 Vault 용량
- 장기 프로젝트화

목표:

- 탐사를 실무 자산으로 전환
- 지속 사용과 실행 가치 회수

---

## 비로그인 사용자에게 정확히 무엇을 열어야 하는가

### 반드시 열어야 하는 것

- 오늘의 광맥 3개 또는 demo 광맥 3개
- 각 광맥의 핵심 조합과 차이
- 광맥 1개 선택 경험
- 결과 샘플 3개
- "이 광맥이 어떤 방향의 아이디어를 여는지"를 보여주는 짧은 해석

### 로그인 전에는 막아야 하는 것

- 결과 저장
- 전체 결과 10개 모두 보기
- outline 생성
- 재방문 가능한 history
- 반복 reroll
- Vault / Lab 진입

### 유료 전용으로 남겨야 하는 것

- 더 깊은 outline
- appraisal depth 확장
- full overview / design / blueprint / roadmap
- 더 높은 사용량
- 고급 비교 및 실행 도구

---

## 권장 게이트 위치

게이트는 체험 전에 두지 않고, `가치를 체감한 직후`에 둔다.

권장 로그인 게이트:

1. `Save to Vault`
2. `See all results`
3. `Generate outline`
4. `Re-scan again`
5. `Continue this idea later`

권장 유료 게이트:

1. `More rerolls`
2. `More generations`
3. `Precise appraisal`
4. `Full overview`
5. `Execution documents`

이 구조의 장점은 사용자가 먼저 가치를 느끼고, 그 다음에 잠금 해제를 이해하게 된다는 점이다.

---

## 비로그인 경험은 실시간 생성보다 `bounded demo`가 낫다

비로그인 첫 경험은 가능하면 실시간 풀 생성보다 `curated demo`, `precomputed result`, `cache-first preview` 구조가 더 좋다.

이유:

- 비용을 통제할 수 있다.
- 속도가 빠르다.
- 체험 품질을 일정하게 유지할 수 있다.
- 콘텐츠 표면으로 재활용하기 쉽다.

권장 구현 원칙:

- demo veins는 제품 팀이 큐레이션한 고품질 조합 사용
- demo ideas는 3개로 제한
- 로그인 이후 real Mine으로 넘어가면 실제 사용자별 광맥과 결과 사용

즉, 비로그인 구간은 "실제 서비스의 축소판"이라기보다 "가장 좋은 첫 장면"에 가깝게 설계한다.

---

## 콘텐츠 전략과의 연결

이 접근 전략은 콘텐츠 전략과도 잘 맞는다.

### 1. 검색 가능한 진입면을 만든다

비로그인 체험면은 검색과 공유가 가능한 표면이 된다. 공개 광맥, 샘플 결과, 오늘의 아이디어는 SEO와 공유 소재로 쓰기 좋다.

### 2. 랜딩의 CTA가 강해진다

랜딩에서 `Sign up`보다 `Try today's vein` 또는 `Explore a live signal` 같은 CTA가 더 설득력 있어진다.

### 3. shareable artifacts를 만들 수 있다

사용자가 로그인 전에도 "이 조합 재미있다"를 느끼면 공유 동기가 생긴다. 이후 공개 permalink, explore page, 오늘의 광맥 카드 등으로 확장 가능하다.

### 4. 검색형과 공유형 콘텐츠를 동시에 지원한다

검색형:

- AI x 시장 x 타깃 조합 페이지
- 특정 도메인/가치 조합 탐색 페이지

공유형:

- 오늘의 광맥
- 오늘의 아이디어
- "이 조합에서 이런 사업 아이디어가 나온다" 콘텐츠

---

## 가격 전략과의 연결

이 접근 전략은 가격 구조를 더 자연스럽게 만든다.

### 비로그인

- 목적: 가치 체감의 시작
- 역할: 가입 이유 만들기

### 무료 가입

- 목적: 반복 가능한 기본 사용
- 역할: 저장, 재방문, 초기 자산화

### Lite / Pro

- 목적: 깊이, 빈도, 실행 가능성, 자산화 확대
- 역할: 사용량과 문서 가치 회수

이 구조가 되면 가격 메시지도 명확해진다.

- 비로그인: "먼저 맛보세요"
- 무료 가입: "이제 저장하고 다시 이어가세요"
- 유료: "이제 더 깊게 만들고 실행 단계까지 가세요"

---

## KPI 관점에서 무엇이 달라지는가

이 전략이 맞다면, 앞으로 봐야 할 초기 핵심 지표는 다음과 같다.

### Public / Anonymous

- 랜딩 -> 체험 진입률
- 공개 광맥 클릭률
- 비로그인 첫 결과 도달률
- 비로그인 -> 로그인 CTA 클릭률

### Signed-in Activation

- 로그인 이후 첫 real mining 완료율
- 첫 저장률
- 첫 overview 생성률

### Monetization

- free -> lite CTA 클릭률
- lite / pro unlock CTA 클릭률
- soft gate 노출 후 전환률

즉, 로그인 이전 퍼널과 로그인 이후 퍼널을 분리해서 본다.

## 단계별 롤아웃 제안

### Phase 1. Public demo surface

가장 먼저 해야 할 것은 `아무것도 못 해보는 상태`를 끝내는 것이다.

포함:

- 랜딩 CTA를 공개 체험면으로 연결
- demo Mine 공개
- 광맥 3개와 샘플 결과 3개 공개
- 저장/outline/전체 결과는 soft gate

### Phase 2. Signed-in free handoff 강화

공개 체험에서 로그인 후 real Mine으로 자연스럽게 이동하게 만든다.

포함:

- sign-in 이후 return path 처리
- "방금 본 방향과 비슷한 real scan" 메시지
- 가입 직후 첫 real Mine 진입 최적화

### Phase 3. 콘텐츠 확장

공개 체험면을 GTM 자산으로 확장한다.

포함:

- explore page
- idea of the day
- 공유 가능한 public artifact

### Phase 4. pricing optimization

soft gate와 free/paid 경계를 실제 데이터로 튜닝한다.

포함:

- 공개 결과 수 3 vs 5 테스트
- save 게이트 vs outline 게이트 순서 조정
- lite/pro unlock 문구 테스트

## 리스크와 대응

### 리스크 1. 공개 체험이 너무 약해 가입 유도가 안 된다

대응:

- 샘플 결과 질을 높인다
- 공개 광맥 조합을 큐레이션한다
- CTA는 더 명확하게 "저장", "전체 보기", "outline"으로 연결한다

### 리스크 2. 공개 체험이 너무 강해 가입 이유가 약해진다

대응:

- 결과 수 제한
- 저장과 지속성 제한
- real reroll / full result / outline는 로그인 후로 유지

### 리스크 3. 비용이 늘어난다

대응:

- 비로그인 경험은 cache / precomputed 우선
- 실시간 생성은 로그인 후로 집중

### 리스크 4. 제품 구조가 이원화되어 복잡해진다

대응:

- public demo와 real Mine의 시각 언어는 공유
- 데이터 소스만 다르게 하고, UI 문법은 최대한 재사용

---

## 최종 원칙

IDEA MINE v2는 사용자가 계정을 만들기 전에 먼저 제품의 핵심 가치를 느끼게 해야 한다. 하지만 그 체험은 `지속 사용 가능한 무료 제품`이 아니라 `잘 설계된 첫 경험`이어야 한다.

따라서 접근 전략의 최종 원칙은 다음 문장으로 정리한다.

> 공개 사용자는 먼저 탐사의 감각과 결과의 매력을 체험한다.
> 계정 사용자는 그 결과를 저장하고 이어간다.
> 유료 사용자는 그것을 더 깊은 자산과 실행 문서로 확장한다.
