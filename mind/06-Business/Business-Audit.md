## 1. 마케팅 전략 평가 (marketing-ideas)

### 잘한 점

- **세계관 브랜딩이 강력함.** 픽셀 아트 + 광산 메타포는 "Engineering as Marketing" 관점에서 바이럴 포텐셜이 있음. 독특한 비주얼은 그 자체로 마케팅 자산
- **도메인 확보** (ideamineai.com) — 브랜드 네이밍 5가지 해석까지 준비된 건 좋음
- **한/영 동시 지원** — 글로벌 확장 의지는 맞는 방향

### 문제점

**문제 1: Go-to-Market이 없다.**
기획은 "무엇을 만들 것인가"에 집중하고, "어떻게 사람들에게 알릴 것인가"는 거의 부재. 34개 문서 중 마케팅/GTM 전략 문서가 0개였음. Sprint 17까지 계획했지만 "첫 100명 유저를 어디서 데려올 것인가?"에 대한 답이 없었음.

**문제 2: Pre-launch 전략 부재.**
현재 단계에서 해야 할 것들:
- Waitlist + 레퍼럴 — 관심 측정
- Product Hunt 준비 — 론칭 시 트래픽
- 커뮤니티 빌딩 — 초기 유저 확보

이 중 어떤 것도 로드맵에 없었음.

**문제 3: 타겟이 맞는가?**
"AI PM/기획자"라는 타겟은 직관적이지만 시장 크기가 매우 작음. 한국에서 "AI PM"이라는 직함을 가진 사람이 몇 명이나 될까? 창업 준비자까지 확장해도 "AI 기술 조합으로 아이디어를 만들고 싶다"는 니즈가 검증되지 않았음.

**평가: 7/10** — 제품 기획은 훌륭하지만, 마케팅 전략은 0점에 가까움.

## 2. 행동심리 분석 (marketing-psychology)

### 잘 활용하고 있는 심리 모델

| 심리 모델 | 적용 방식 | 평가 |
|-----------|----------|------|
| Endowment Effect | 금고에 아이디어를 쌓으면 애착 형성 | 좋음 |
| Goal-Gradient Effect | 레벨, 배지, 연속 접속 -> 완료에 가까울수록 동기 상승 | 좋음 |
| IKEA Effect | 키워드 직접 조합 -> "내가 만든 아이디어" 가치 상승 | 좋음 |
| Zero-Price Effect | 무료 5원석 vs 유료 6원석 -> 무료 진입 장벽 낮춤 | 좋음 |
| Scarcity | 희귀 광맥, 고대 동전 랜덤 드롭 | 적절 |

### 놓치고 있거나 위험한 심리 모델

**위험 1: Jobs-to-Be-Done 혼란**
사용자의 JTBD는 "AI 사업 아이디어를 빠르게 만들고 싶다"인데, 제품은 게이미피케이션에 치우쳐 핵심 job을 흐리고 있음. 배지, 레벨, 핑크 결정, 고대 동전, 꾸미기 상점... 이건 "기획 도구"인가 "모바일 게임"인가?

"일처럼 쓰는 게임, 게임처럼 들어오는 기획 툴"이라는 슬로건은 멋지지만, 실제로 두 마리 토끼를 잡은 제품은 거의 없다.

**위험 2: Paradox of Choice (선택의 역설)**
아이디어 10개를 한 번에 생성하는데, 이게 "풍성함"이 아니라 "어떤 게 좋은지 모르겠다"가 될 수 있음. 10개 중 3개가 비슷하면 사용자는 피로를 느낌. KPI 문서에서도 "10개 vs 3개 고품질"을 고민하고 있는데 — 3개 고품질이 맞을 확률이 높음.

**위험 3: Sunk Cost + 게임 피로**
3통화 시스템(핑크 결정 + Pink Diamond + 고대 동전)은 모바일 RPG 설계. 기획자/창업자 타겟에게 이 복잡한 재화 시스템이 "내가 이걸 왜 쓰고 있지?" 순간을 만들 수 있음. 첫 채굴의 재미가 사라진 후 돌아올 이유가 "재화 모으기"라면 위험.

**위험 4: Activation Energy가 높음**
온보딩 8단계(스플래시 -> 세계관 -> 가이드 -> 광맥 -> 리롤 -> 생성 -> 저장 -> 업셀)는 BJ Fogg 모델 관점에서 너무 길다. "30초 안에 보상" 원칙을 세웠지만, 실제로 8단계를 30초에 끝내기 어려움.

**평가: 6/10** — 축적/애착 심리는 잘 설계했지만, 핵심 JTBD를 게임이 잡아먹을 위험이 큼.

## 3. 가격/티어 평가 (pricing-strategy)

### 현재 구조 분석

| 항목 | 현재 설계 | 평가 |
|------|----------|------|
| 티어 수 | Free / Lite / Pro 3단계 | Lite 정체가 불명확 |
| 가치 지표(Value Metric) | AI 키워드 슬롯 해금 | 영리한 선택 |
| 무료 체험 | 5원석 조합 + 일 1회 생성 | 너무 제한적이거나 너무 충분하거나 |
| 업셀 포인트 | AI 슬롯 잠금 보여주기 | 좋은 설계 |
| 가격 리서치 | 없음 | 치명적 부재 |

### 문제점

**문제 1: 가격이 정해지지 않았다.**
`Tier-Structure`에 Free/Lite/Pro 기능 차이는 있지만 실제 가격($X/월)이 없음. 가격 결정을 위한 경쟁사 분석, WTP(Willingness-to-Pay) 리서치, Van Westendorp 조사가 전혀 없음.

**문제 2: Value Metric의 위험**
"AI 카테고리 해금"이 가치 지표인데, 무료 유저가 5원석으로 생성한 아이디어와 유료 6원석 아이디어의 품질 차이가 체감 가능해야 업셀이 작동함. 만약 5원석 결과도 "꽤 괜찮다"면 유료 전환 동기가 없음. 반대로 5원석이 "쓸모없다"면 유저가 이탈함. 이 밸런스가 제품의 생사를 결정하는데, 검증 방법이 없음.

**문제 3: 4축 수익 구조는 과도함**
광고 + 인앱 결제 + 구독 + B2B 멤버십. PMF 전에 4가지 수익 모델을 설계하는 건 전형적인 과잉 설계. First Principles로 생각하면: "첫 $1을 벌 방법은 무엇인가?"에만 집중해야 함.

**문제 4: Lite 티어의 존재 이유**
`Tier-Structure`에 Free/Lite/Pro를 언급하지만 Lite의 구체적 차별점이 부족. Decoy Effect를 쓰려면 Lite가 "Pro로 가게 만드는 미끼"여야 하는데, 그런 설계가 보이지 않음. 3개 티어를 운영할 리소스가 있는지도 의문.

**평가: 4/10** — AI 슬롯 해금이라는 가치 지표는 영리하지만, 가격 리서치 부재와 과잉 수익 모델이 문제.

## 4. 전환율(CRO) 관점 평가 (page-cro)

### 앱 내 전환 퍼널

```
온보딩 -> 첫 채굴 -> 아이디어 저장 -> 개요서 생성 -> Pro 전환
```

KPI 문서에서 정의한 지표들은 적절함. 다만:

**문제 1: 5초 테스트 실패 위험**
"이 앱이 뭔지 5초 안에 이해할 수 있는가?" — 광산 세계관이 아름답지만, 처음 들어온 사용자가 "이건 AI 아이디어 생성 도구입니다"를 즉시 이해할 수 있을까? "광맥", "원석", "채굴"... 기획자에게 이 용어가 직관적인가, 아니면 학습이 필요한가?

CRO 관점에서 가장 위험한 것은 메타포가 명확성을 해치는 순간. "키워드 조합 -> 아이디어 생성"이라고 말하면 3초에 이해하는 걸, "광맥 선택 -> 원석 채굴"이라고 하면 10초+가 필요.

**문제 2: 핵심 CTA가 불명확**
홈 화면 구성요소가 7개(상태바, 광맥 3개, 리롤, 퀘스트, 주말 배너, 금고 미리보기, 예고 구역). Hick's Law 위반. 처음 온 사용자에게 명확한 하나의 행동("이 광맥을 선택하세요")이 즉시 보여야 하는데, 정보가 너무 많음.

**문제 3: 랜딩 페이지(ideamineai.com) 전략 부재**
앱스토어 전 웹 랜딩이 첫 접점인데, 어떤 메시지로 어떻게 전환시킬지 계획이 없음. 헤드라인, 소셜 프루프, CTA 전략이 필요.

**평가: 5/10** — 퍼널 구조는 있으나, 메타포가 명확성을 해칠 위험 + 홈 화면 정보 과부하.

## 5. SEO 전략 평가 (seo-audit)

### 현재 상태

- 도메인: ideamineai.com (확보 완료)
- 웹사이트: 아직 없음 (개발 전)
- 블로그/콘텐츠: 없음

### 문제점

**문제 1: 앱 퍼스트 = SEO 난이도 극상**
IDEA MINE은 모바일 앱 중심 제품. SEO는 웹 콘텐츠에서 시작되는데, 앱의 핵심 가치(AI 아이디어 생성)를 웹에서 어떻게 보여줄 것인가? "Free AI idea generator" 같은 키워드는 이미 레드오션.

**문제 2: 검색 의도(Search Intent) 불일치**
"AI business idea generator"를 검색하는 사람은 즉시 결과를 원함. 앱 설치를 거치는 건 마찰이 큼. 웹에서 무료 체험판(조합 1회 -> 결과 3개)을 제공하는 Engineering-as-Marketing 전략이 없으면 SEO 트래픽을 전환시키기 어려움.

**문제 3: 콘텐츠 SEO 계획 부재**
글로벌 확장을 목표로 한다면 아래가 필요하지만 로드맵에 없음:
- 블로그 (AI 트렌드, 창업 아이디어 사례)
- Programmatic SEO (키워드 조합별 아이디어 랜딩)
- Glossary (AI 기술 용어 사전)

**기회:**
키워드 택소노미(109개)를 활용한 프로그래매틱 SEO는 잠재력이 큼. "AI + 헬스케어 + 구독 모델 아이디어" 같은 롱테일 키워드 페이지를 자동 생성하면 수천 개의 랜딩 페이지를 만들 수 있음.

**평가: 3/10** — SEO 전략이 아예 없음. 앱 퍼스트 구조에서 오가닉 트래픽 확보 계획이 필요.

## 6. 소셜 미디어 전략 평가 (social-content)

### 현재 상태: 소셜 전략 부재

### 기회와 위험

**기회: 비주얼 바이럴 포텐셜**
픽셀 아트 스타일은 소셜에서 눈에 띄는 차별화 요소. 특히:
- 트위터/X: AI 커뮤니티가 활발, 픽셀 아트 스크린샷이 리트윗 유도
- 인스타그램: 레트로 감성 릴스
- 프로덕트헌트: 유니크한 비주얼은 어워드 가능성 높음

**위험 1: 타겟과 톤의 불일치**
AI PM/기획자는 LinkedIn/트위터 헤비 유저. 이들의 피드에서 "픽셀 게임"이 들어오면 "장난감 아닌가?"라는 첫인상 리스크가 있음. B2B 도구의 신뢰감과 게임의 가벼움을 동시에 전달하는 것은 매우 어려운 커뮤니케이션 과제.

**위험 2: 1인 개발 + 소셜 운영 = 번아웃**
소셜 콘텐츠는 최소 주 3-5회 포스팅 + 일 30분 인게이지먼트가 필요. 개발하면서 이걸 병행하기 극도로 어려움.

**권장 전략: Build in Public**
1인 개발의 장점은 "빌딩 인 퍼블릭" 콘텐츠. 트위터에서 개발 과정을 공유하고, 픽셀 에셋 제작 과정을 릴스로 올리는 것이 제품 출시 전 가장 효율적인 전략.

| 플랫폼 | 콘텐츠 필러 | 빈도 |
|--------|------------|------|
| 트위터/X | 개발 일지, AI 인사이트 | 주 3-5회 |
| LinkedIn | 기획 프레임워크, 창업 인사이트 | 주 2-3회 |
| 인스타그램 | 픽셀 아트 제작 과정 | 주 1-2회 |

**평가: 4/10** — 비주얼 차별화 포텐셜은 높지만, 소셜 전략 부재 + 타겟-톤 불일치 리스크.

## 종합 평가

### 점수 요약

| 영역 | 점수 | 한 줄 평가 |
|------|------|-----------|
| 제품 기획 | 8/10 | 세계관, 루프 설계, 기능 정의 매우 우수 |
| 마케팅 전략 | 2/10 | GTM 전략 거의 부재 |
| 행동심리 설계 | 6/10 | 축적 심리 좋으나 JTBD 혼란 위험 |
| 가격/티어 | 4/10 | 가치 지표 좋으나 가격 리서치 없음 |
| 전환율(CRO) | 5/10 | 퍼널 있으나 메타포가 명확성 해칠 위험 |
| SEO | 3/10 | 전략 부재, 앱 퍼스트의 본질적 한계 |
| 소셜 미디어 | 4/10 | 비주얼 포텐셜 있으나 전략 부재 |
| **종합** | **4.6/10** | |

### 핵심 리스크 TOP 5

1. **검증 전 과잉 설계.** 17개 스프린트, 3개 통화, 4개 수익 모델. 핵심 가설("사람들이 이 조합형 생성을 유용하다고 느끼는가?")이 검증 안 된 상태에서 Phase 4 생태계까지 설계한 건 위험. 이건 Map =/= Territory — 기획서가 현실을 대체하고 있음.

2. **경쟁 분석 부재.** ChatGPT에게 "AI x 헬스케어 x 구독 모델 아이디어 10개 알려줘"라고 치면 무료로 비슷한 결과를 얻음. "그냥 GPT에 물어보면 되는 거 아니야?"에 대한 방어 논리가 문서 어디에도 없었음.

3. **게임이 도구를 잡아먹을 위험.** 기획자에게 레벨업, 배지, 핑크 결정, 고대 동전이 과연 동기 부여가 될까? 기획자는 결과물의 품질에 돈을 내지, 캐릭터 꾸미기에 돈을 내지 않을 가능성이 높음.

4. **1인 개발 x 거대한 스코프.** Phase 1A만 해도 7 스프린트 x 픽셀 에셋 15+종. React 처음인 상태에서 이 분량은 현실적으로 수개월. 그 사이 시장이 변할 수 있음.

5. **고객 확보 채널 부재.** 제품을 만들어도 사용자를 어디서 데려올지 계획이 없음. "빌드하면 올 것이다(Build it and they will come)"는 가장 흔한 인디 해커 실패 패턴.
