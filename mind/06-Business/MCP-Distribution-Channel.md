# MCP Distribution Channel

> IDEA MINE MCP 서버는 제품이 아니라 **영구 무료 획득 채널**. MCP에서 맛보기 -> 앱으로 유도하는 퍼널의 입구.

---

## 핵심 포지셔닝

### MCP는 "채널"이다, "제품"이 아니다

```
MCP (입구)              앱 (본체)
├── 맛보기 체험          ├── 저장/축적
├── 휘발성 결과          ├── 금고/히스토리
├── 로그인 불필요        ├── 게이미피케이션
└── 텍스트 인터페이스    └── 픽셀 아트 세계관
```

MCP를 "제품"으로 만들면 픽셀 아트, 세계관, Playable Tool이라는 핵심 차별점이 전부 사라짐. MCP는 "키워드 조합 -> 아이디어 생성"의 핵심 가치만 빠르게 체험시키고, 풀 경험은 앱으로 유도하는 역할.

---

## 왜 MCP 채널이 강력한가

### 마찰 제로 획득

```
기존:  검색 -> 랜딩 -> 앱스토어 -> 설치 -> 회원가입 -> 첫 채굴 (6단계)
MCP:   MCP 설치 -> 바로 채굴 (2단계)
```

### 심리적 전환 흐름

```
MCP 설치 (10초, 무료)
-> 키워드 조합 시도 (Foot-in-the-Door)
-> 아이디어 10개 생성 (가치 체험)
-> "이걸 저장하고 싶다" (Endowment Effect)
-> "개요서도 더 만들고 싶다" (Goal-Gradient)
-> 앱/웹으로 이동 (자연스러운 전환)
```

### 배포 채널 다양화

MCP 마켓플레이스 자체가 새로운 발견 경로:
- Claude Code 사용자가 "idea generator MCP" 검색
- Cursor/Windsurf 등 AI 도구 확장
- MCP 디렉토리/큐레이션 사이트

### "ChatGPT로 해도 되잖아?"에 대한 직접 답변

"저희 MCP를 깔아보세요. /mine 치면 6차원 조합이 바로 나옵니다. ChatGPT에 프롬프트 쓰는 것보다 빠르죠?"

---

## 타겟 사용자

### 1차: AI 코딩 도구 사용자

Claude Code, Cursor 등에서 코딩하다가 "다음에 뭘 만들지?" 떠올릴 때 바로 사용. 개발자/메이커가 주 타겟.

### 2차: AI 채팅 사용자

Claude Desktop 등 비개발자 포함. 대화하다가 사업 아이디어가 필요할 때. 1차 타겟 안정화 후 확장.

---

## MCP 기능 범위

### 제공하는 것 (앱 Free 티어와 동일)

| 기능 | MCP | 앱 Free |
|------|-----|---------|
| 오늘의 광맥 3개 | 동일 | 동일 |
| 리롤 2회 | 동일 | 동일 |
| 채굴 1회/일 -> 원석 10개 | 동일 | 동일 |
| 개요서 1회 | 동일 | 동일 |
| 키워드 3~5개 사용 | 동일 | 동일 |
| AI 키워드 | 잠금 (5원석) | 잠금 (5원석) |

### 앱에서만 가능한 것 (MCP에서 유도)

| 기능 | MCP에서 메시지 | 앱에서 |
|------|-------------|--------|
| 저장/금고 | "이 아이디어를 저장하려면 ideamineai.com에서" | 가능 |
| 감정 | "이 아이디어의 시장성을 평가하려면 앱에서" | 가능 |
| 히스토리 | "지난 채굴 결과를 보려면 앱에서" | 가능 |
| 추가 채굴 | "오늘 채굴을 이미 했어요. 앱에서 더 채굴할 수 있어요" | Lite/Pro |
| 풀 조합 | "모든 키워드 사용은 Pro에서" | Pro |
| AI 키워드 | "AI 키워드 해금은 Lite/Pro에서" | Lite/Pro |

### 핵심 유도 포인트

MCP에서 앱으로 전환하는 결정적 순간은 **"저장이 안 되는 것"**:

```
사용자가 MCP에서 좋은 아이디어를 발견
-> "이거 저장하고 싶다"
-> MCP: 결과가 휘발성 (저장 불가)
-> "앱에서는 금고에 쌓을 수 있어요"
-> 앱 설치/웹 가입
```

횟수 제한은 "짜증"을 만들지만, 휘발성은 "아쉬움"을 만듦. 아쉬움이 전환 동기로 훨씬 강함 (Loss Aversion).

## MCP-앱 통합 로직

### 계정 없이 동작

MCP는 로그인/계정 없이 동작. 일일 제한은 디바이스/IP 기반 소프트 리밋.

### 비용 통제 + 트래킹

- 1유저 하루 최대: 채굴 1회($0.01-0.03) + 개요서 1회($0.02-0.05) = ~$0.05
- 시간당 rate limit으로 악용 방지 (예: 시간당 10회)
- 앱 Free 티어와 동일한 비용 구조이므로 별도 비용 모델 불필요

**API source 태그로 비용 분리 추적:**

모든 API 요청에 `source` 필드 포함:
```
앱:  { ..., source: "app" }
웹:  { ..., source: "web" }
MCP: { ..., source: "mcp" }
```

Supabase `api_logs` 테이블에 기록:
```
id | endpoint | source | ip | created_at | cost_estimate
```

이러면 "MCP 채널 월 $30, 앱 월 $20" 같은 채널별 비용 분석 가능. MCP 비용이 전환 유저 대비 과도하면 rate limit 조정.

### 향후 계정 연동 (Phase 2+)

앱 계정이 있는 유저가 MCP에서 로그인하면:
- MCP 채굴 결과가 앱 금고에 자동 저장
- Lite/Pro 혜택이 MCP에서도 적용 (추가 채굴, AI 키워드 등)
- MCP가 "앱의 CLI 버전"이 됨

## 비즈니스 임팩트

### GTM 채널로서의 가치

| 기존 GTM 채널 | MCP 추가 시 |
|-------------|------------|
| Waitlist -> 앱 설치 (마찰 높음) | MCP 설치 = 1분 (마찰 제로) |
| PH/Reddit으로 인지 | MCP 마켓플레이스에서 발견 |
| "첫 100명" 시딩 어려움 | AI 도구 유저가 자연 유입 |
| 개발자 타겟 접근 어려움 | 개발자가 쓰는 도구 안에 존재 |

### 소셜 바이럴

"IDEA MINE MCP 깔았는데 Claude Code에서 사업 아이디어 바로 나옴" -> 개발자 커뮤니티 바이럴. BIP 콘텐츠와 시너지.

## 개발 우선순위

### Phase 0에서 MCP 먼저 만들 수도 있음

```
기존 Phase 0:  프로토타입(Expo 앱) -> 테스터 공유
대안 Phase 0:  MCP 먼저(1일) -> 개발자에게 공유 -> 반응 보고 앱 결정
```

MCP가 Expo 앱보다 빠르게 만들 수 있고, 타겟(아이디어가 필요한 개발자)에게 직접 닿음. Phase 0 검증 도구로도 활용 가능하지만, 주 역할은 영구 무료 획득 채널.
