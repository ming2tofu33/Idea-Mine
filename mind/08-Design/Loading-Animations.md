# Loading Animations

> 모든 로딩 상황에 세계관에 맞는 고유 연출을 적용한다.
> 기본 스피너(ActivityIndicator)는 사용하지 않는다.
> 구현 상세는 `plans/2026-03-23-loading-animations.md` 참조.

---

## 설계 원칙

- **세계관 일관성**: 모든 로딩은 광산 세계관의 행위에 대응
- **시간 비례 복잡도**: 짧은 로딩은 단순, 긴 로딩은 단계적 진행감
- **정직한 진행감**: 가짜 진행률 바 대신, 행위 자체의 변화로 진행을 표현
- **최소 표시 시간**: 짧은 로딩(1~2초)은 최소 1.5초, 앱 시작은 최소 2초 — 연출이 깜빡거리지 않도록

---

## 전체 로딩 맵

| 상황 | 세계관 의미 | 대기 시간 | 연출 핵심 | 상태 |
|------|------------|----------|----------|------|
| 앱 시작 | 광산에 입장 | 1~2초 | 어둠 → 랜턴 점등 → 로고 등장 | 구현 완료 |
| 광맥 불러오기 | 벽을 탐색 | 1~2초 | 랜턴이 좌우로 벽을 비춤 | 구현 완료 |
| 채굴 중 | 곡괭이로 캐기 | 5~15초 | 곡괭이 내리치기 + 파편 + 광차에 원석 | 구현 완료 |
| 리롤 | 광맥 폭파 교체 | 1~2초 | 흔들림 → 폭발 → 먼지 → 새 광맥 | 구현 완료 |
| 금고 반입 | 광차로 운반 | 1~2초 | 원석 담기 → 광차 레일 이동 | 구현 완료 |
| 로그인 전 | 출입증 발급 | 2~3초 | 카드 등장 + APPROVED 도장 | 구현 완료 |
| 로그인 후 | 신원 확인 | 2.5초 | 출입증 스캔 라인 왕복 | 구현 완료 |
| 개요서 생성 | 원석 연마 + 문서화 | 20~40초 | 3단계 연출 (연마→전환→개요서 틀) | 구현 완료 |

## 1. 앱 시작 — 랜턴 입장

**컴포넌트**: `LanternEntry`

어둠 속에서 랜턴 불빛이 켜지고, 빛이 퍼지면서 "IDEA MINE" 로고가 골드로 드러나는 연출.

```
[0.0초] 완전 어두운 화면 (#000)
[0.5초] 랜턴(🏮) 등장 + 원형 glow 퍼짐 (goldGlow)
[1.0초] "IDEA MINE" 로고 골드로 페이드인
[1.2초] "광산에 입장하는 중..." 페이드인
[완료]  메인 화면으로 전환
```

---

## 2. 광맥 불러오기 — 랜턴 벽 탐색

**컴포넌트**: `LanternScan`

랜턴 불빛이 가운데를 기준으로 좌우 25%씩 왔다갔다하며 벽을 비추는 연출.

```
[시작] 랜턴(🏮) + 타원형 glow가 좌↔우 반복 이동
       "벽을 살피는 중..."
[완료] 광맥 카드 페이드인
```

- 이동 범위: `Math.min(width, 430) * 0.25` (반응형)
- 이동 속도: 1.5초/편도, ease-in-out
- 랜턴 위치: 화면 중앙 기준 (left: 50%, marginLeft: -40)

---

## 3. 채굴 중 — 곡괭이 + 광차

**컴포넌트**: `MiningLoader`

곡괭이가 광맥을 내리치고, 나온 원석이 광차에 담기는 3단계 연출.

```
[1단계: 스캔, ~4초]
   곡괭이(⛏) 반복 내리침 (rotation 0→-45→+15→0, 1.35초 사이클)
   골드 파편(✦) 튀어오름 (내리치기 동기화)
   광맥: 큰 크기, 광차에 원석 1개
   "광맥을 스캔하는 중..."

[2단계: 추출, ~4초]
   광맥 줄어듦, 광차에 원석 2개
   "결정 구조를 분석하는 중..."

[3단계: 완료, ~4초+]
   광맥 작아짐, 광차에 원석 3개
   "아이디어 결정을 추출하는 중..."
```

---

## 4. 리롤 — 광맥 폭파 교체

**컴포넌트**: `RerollBlast`

기존 광맥이 흔들리고 폭발한 뒤 새 광맥이 등장하는 인라인 연출.

```
[0.0초] 기존 카드 좌우 흔들림 (±2px, 3회)
[0.3초] 축소 (scale 0.95)
[0.4초] 💥 폭발 + 골드/오렌지 파편 8개 사방 확산 + Haptic
[0.6초] 회색 먼지 퍼짐 → 걷힘
[1.0초] 새 광맥 카드 등장 (scale 0.8→1 + fade in)
[1.2초] 완전히 안착
```

- 전체 1.2초, 짧고 시원한 느낌
- `absoluteFillObject`로 부모 기준 상대 배치 (반응형 OK)

---

## 5. 금고 반입 — 광차 출발

**컴포넌트**: `MinecartDepart`

선택한 원석이 광차에 담기고, 레일을 타고 화면 끝까지 이동하는 연출.

```
[0.0초] 레일(나무색 #4A3728) + 광차(화면 왼쪽)
        원석(💎)이 이미 광차에 담겨 있음
        "금고로 운반하는 중..." 텍스트 페이드인
[0.2초] 광차가 왼쪽→오른쪽 끝까지 이동 (1.2초, ease-in-out)
        이동 중 덜컹거림 (translateY ±2px 반복)
[1.4초] 광차 화면 밖으로 사라짐
```

- 이동 거리: `Math.min(width, 430)` (반응형)
- 원석 개수: 실제 선택 개수 반영 (최대 5개)

---

## 6. 로그인 — 출입증 스캔

**컴포넌트**: `IdCardScan`

두 가지 variant로 로그인 전/후를 구분한다.

### variant="preparing" (로그인 전, 버튼 누른 후)

출입증 카드가 만들어지는 연출.

```
[0.0초] 카드 페이드인 + 스케일업 (opacity 0→1, scale 0.8→1, 600ms)
        카드: IDEA MINE 로고 + ⛏ + "출입증 발급"
        테두리: text.muted (회색 — 아직 미승인)
[1.0초] "APPROVED" 도장이 우하단에 찍힘 (opacity 0→1, -12도 회전)
        도장 테두리: accent.gold
        "출입증 준비 중..."
```

### variant="scanning" (로그인 후, OAuth 복귀)

출입증을 스캔해서 신원을 확인하는 연출.

```
[시작] 카드: IDEA MINE 로고 + ⛏ + "광부 출입증"
       테두리: accent.gold (승인됨)
       골드 스캔 라인이 카드 위→아래 왕복 (800ms/편도, linear)
       라인 뒤 glow 잔상 (shadowRadius 6)
       "광부 신원 확인 중..."
[2.5초] 스캔 완료 → 메인 화면 이동
```

## 7. 개요서 생성 — 원석 연마 + 개요서 틀

**컴포넌트**: `LabLoader`

가장 긴 로딩(20~40초). 3단계 연출로 지루함을 최소화한다.

### Phase A: 원석 연마 (0~8.5초)

민트 블루 실험실 배경에서 거친 원석이 4단계를 거쳐 빛나는 보석으로 변하는 연출.

```
화면 구성:
   [유리관]              [램프]
       ┌─────────────┐
       │   glow(원)   │
       │   ┌─────┐   │
       │   │ 원석  │   │
       │   └─────┘   │
       └─────────────┘
    ┌──── 작업대(민트) ────┐
    └──────┤ 다리 ├──────┘

단계:
[0초] 거친 원석 — 각진 사각형, #4A4E62, glow 없음, 램프 꺼짐
[3초] 1차 연마 — 둥근 사각형, #5A5E72, 램프 점등
[6초] 2차 연마 — 다이아몬드형(45도), #A8E6CF, glow 등장, ✨ 스파클 시작
[8초] 완성 보석 — 골드(#C4B07A), 강한 glow, ✨ pulse 계속
```

| 단계 | borderRadius | rotation | scale | 색상 | glow | 스파클 |
|------|-------------|----------|-------|------|------|--------|
| 거친 원석 | 0 | 0deg | 1.0 | #4A4E62 | 0 | 없음 |
| 1차 연마 | 8 | 0deg | 0.95 | #5A5E72 | 0 | 없음 |
| 2차 연마 | 4 | 45deg | 0.7 | #A8E6CF | 0.3 | ✨ |
| 완성 보석 | 4 | 45deg | 0.7 | #C4B07A | 0.6 | ✨ |

### Phase B: 전환 (8.5~10초)

보석이 위로 떠오르며 실험실 씬이 사라지고 문서 화면으로 전환.

```
[8.5초] 보석 위로 떠오름 (translateY -20, 500ms)
[9.0초] 전체 페이드아웃 + 보석 위로 사라짐 (translateY -200, 600ms)
[9.6초] Phase C 페이드인
```

### Phase C: 개요서 작성 (10초~완료)

빈 개요서 틀이 순서대로 나타나고, shimmer 로딩으로 "작성 중"을 표현.

```
[10초] 개요서 카드 틀 등장 (💎 + "프로젝트 개요서")
[13초] "문제 정의" 섹션 등장 + shimmer
[17초] "타깃 사용자" 섹션 등장
[21초] "핵심 기능" 섹션 등장
[25초] "수익 구조" 섹션 등장
[28초] "읽기 좋은 문서로 다듬는 중이에요"
[32초] 대기 모드 진입 — 순환 메시지 + 점 애니메이션
```

### 대기 모드 (32초~API 완료)

모든 섹션 등장 이후 API 응답을 기다리는 동안, 멈춘 느낌을 방지하는 연출.

**메시지 순환** (5초 간격):
- "거의 다 됐어요"
- "마지막 문단을 정리하는 중이에요"
- "조금만 더 기다려주세요"
- "문서를 꼼꼼히 검토하는 중이에요"
- "마무리 작업 중이에요"

**점 애니메이션** (0.5초 간격):
각 메시지 끝에 `.` → `..` → `...` → `` 반복

shimmer도 계속 동작하여 "살아있는" 느낌 유지.

## v2 전환 노트

v1에서 구현된 로딩 연출들(LanternEntry, MiningLoader, RerollBlast 등)은 Expo/React Native 기반이다.
v2에서는 Framer Motion + CSS/Canvas 기반으로 재구현하되, **세계관 연출 개념(광맥 탐색, 원석 연마 등)은 유지**한다.
단, 프리미엄 스페이스 UI 방향에 맞게 이모지 대신 정교한 모션 그래픽으로 격상.
