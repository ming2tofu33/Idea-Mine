# Space Language

> v2의 `S0-DIR-01` 기준 문서. Mine / Vault / Lab / Basecamp가 각각 어떤 감정, 위계, 배경, 상호작용 톤을 가져야 하는지 정리한다.

---

## 왜 이 문서가 필요한가

`Premium-Space-UI-Direction`은 전체 디자인 테제를 정의한다.  
이 문서는 그 상위 방향을 실제 공간 단위로 풀어서, 화면을 만들 때 바로 참고할 수 있는 **공간별 언어 사전** 역할을 한다.

즉:

- [[Premium-Space-UI-Direction]] = 전체 세계의 스타일
- [[Space-Language]] = 각 공간이 어떻게 다르게 느껴져야 하는지

---

## 공통 원칙

모든 공간은 같은 우주 안에 있어야 한다. 하지만 같은 화면처럼 느껴지면 안 된다.

- 공통 기반: 깊은 배경, 정밀한 장비감, 절제된 신호 색
- 차이 요소: 여백, 밀도, 글로우 강도, 정보 위계, 상호작용 리듬
- 기준: 분위기는 기능을 덮지 않고, 기능 이해를 더 빠르게 만들어야 한다

---

## S0-DIR-01 작업 체크리스트

### 1. 공통 차이축 확정

- [ ] 여백
- [ ] 밀도
- [ ] 글로우 강도
- [ ] 정보 위계
- [ ] 상호작용 리듬

### 2. 공간별 산출물 확정

각 공간마다 아래 4가지를 남긴다.

- 한 줄 정의
- 블록 구조
- do / don't
- 3초 판별 질문

### 3. 완료 기준

- [ ] Mine / Vault / Lab / Basecamp를 각 1문장으로 설명할 수 있다
- [ ] 4개 공간의 블록 구조를 이름으로 나열할 수 있다
- [ ] 4개 공간의 do / don't가 각각 3개 이상 있다
- [ ] 다음 단계인 `S0-DS-01`, `S0-SHELL-01`이 이 문서만 보고 시작 가능하다

## Space Snapshot

| Space | 핵심 감정 | 첫 인상 | UI 키워드 |
|------|----------|--------|----------|
| Mine | 경이감, 탐사, 선택의 긴장감 | "오늘의 신호를 발견했다" | cinematic, deep, luminous |
| Vault | 신뢰감, 축적감, 재방문 | "내가 모아둔 자산이 남아 있다" | archive, structured, calm |
| Lab | 집중감, 분석, 판단 | "이 자산을 이해하고 다듬는다" | precise, clean, tool-like |
| Basecamp | 정체성, 복귀점, 운영 허브 | "여기가 내 기준점이다" | grounded, quiet, personal |

## The Mine

### 한 줄 정의

오늘의 광맥을 고르고 바로 채굴을 시작하게 만드는 flagship 진입 공간.

### 역할

- 오늘의 광맥을 고르고
- 채굴을 시작하고
- 첫 선택을 내리는 공간

### 감정 목표

- 가장 cinematic해야 한다
- 가장 먼저 "들어가고 싶다"가 느껴져야 한다
- 하지만 복잡한 게임처럼 보이면 안 된다

### 비주얼 언어

- 넓은 여백
- 깊은 성운/신호 레이어
- 가장 강한 글로우와 하이라이트
- 어두운 배경 위에 signal pink, cold cyan이 또렷하게 읽힘

### 정보 위계

1. 오늘의 광맥
2. 채굴 액션
3. 일일 상태 / 제한 정보

### 블록 구조

1. Hero Signal Zone
2. Vein Selection Zone
3. Primary Action Zone
4. Daily State Zone

### 상호작용 톤

- 선택은 또렷하고 약간 의식적이어야 한다
- hover / focus / selected 상태 차이가 분명해야 한다
- 로딩은 "시스템이 신호를 해석 중"처럼 느껴져야 한다

### 하지 말아야 할 것

- 텍스트 정보판처럼 만들기
- 카드 리스트만 나열해서 cinematic 감각을 잃기
- 상태 정보가 광맥보다 먼저 보이게 만들기

### 3초 판별 질문

- 첫 시선이 광맥과 채굴 액션으로 바로 가는가?
- cinematic하지만 실제 선택 행동이 흐려지지 않는가?

## The Vault

### 한 줄 정의

저장된 자산을 차분하게 다시 훑고 다음 행동으로 보내는 아카이브 허브.

### 역할

- 저장된 원석과 문서를 다시 보고
- 다음 행동으로 보내는 허브 역할을 한다

### 감정 목표

- 차분하고 신뢰감 있어야 한다
- "남아 있다", "쌓여 간다"는 감각이 먼저 와야 한다
- Mine보다 조용하지만 빈약하게 느껴지면 안 된다

### 비주얼 언어

- 같은 우주 계열이지만 에너지는 억제
- 구조와 정렬, 그룹핑이 더 중요
- 글로우보다 보관/분류/정리의 인상이 먼저
- 아카이브적 표면, 스캔 가능한 레이아웃

### 정보 위계

1. 저장된 자산 목록
2. 상태 배지와 최근성
3. Lab으로 보내는 다음 행동

### 블록 구조

1. Archive Header Zone
2. Asset List Zone
3. Status / Filter Zone
4. Next Action Zone

### 상호작용 톤

- 빠르게 훑고 고를 수 있어야 한다
- 선택보다는 탐색과 재방문 리듬이 중요하다
- hover/focus는 조용하되 신뢰감 있게 반응

### 하지 말아야 할 것

- Mine처럼 과한 글로우와 연출
- 문서와 원석의 상태가 한눈에 구분되지 않는 구조
- 지나치게 장식적인 금고 판타지

### 3초 판별 질문

- 저장된 자산이 구조적으로 잘 보이는가?
- Mine보다 차분하고 아카이브적으로 느껴지는가?

## The Lab

### 한 줄 정의

선택한 아이디어를 이해하고 평가하고 문서 자산으로 확장하는 분석 작업대.

### 역할

- 선택된 아이디어를 이해하고
- 평가하고
- 문서 자산으로 확장하는 공간

### 감정 목표

- 가장 분석적이고 집중된 공간
- 감정보다 판단과 구조가 먼저 읽혀야 한다
- "여기서는 일할 수 있다"는 감각이 중요하다

### 비주얼 언어

- 가장 깨끗한 계층
- 배경 노이즈 최소
- 도구, 패널, 결과물의 구분이 선명
- cyan / steel 계열 비중 증가, pink는 고우선 선택/CTA에만 사용

### 정보 위계

1. 지금 어떤 자산을 다루는지
2. 현재 단계가 무엇인지
3. 다음 생성/평가 행동
4. 결과 문서의 구조

### 블록 구조

1. Current Asset Zone
2. Step / Progress Zone
3. Primary Action Zone
4. Result Structure Zone

### 상호작용 톤

- 입력, 생성, 확인 흐름이 명확해야 한다
- transition은 빠르고 정밀해야 한다
- "멋진 장면"보다 "도구가 잘 읽힌다"가 우선이다

### 하지 말아야 할 것

- Mine 같은 감정 연출 재사용
- 결과 문서보다 장식이 먼저 보이는 구조
- 단계 흐름이 한눈에 안 보이는 복잡한 패널 배치

### 3초 판별 질문

- 지금 어떤 자산과 어떤 단계가 핵심인지 바로 보이는가?
- 장식보다 판단과 문서 구조가 먼저 읽히는가?

## Basecamp

### 한 줄 정의

프로필과 설정, 계정 상태를 조용하게 정리하는 개인 기준점이자 운영 허브.

### 역할

- 프로필, 설정, 계정 상태의 기준점
- 필요 시 admin 진입을 품는 운영 허브

### 감정 목표

- 조용하고 단정해야 한다
- Mine / Vault / Lab보다 개인적이고 grounded한 공간이어야 한다
- "여기로 돌아와 정리한다"는 느낌이 중요하다

### 비주얼 언어

- 우주적 기반은 유지하되 가장 절제된 장면
- 개인 정보와 설정이 읽기 쉬운 구조
- 장식보다 신뢰감, 정돈감, 소유감이 우선

### 정보 위계

1. 프로필 / 계정 상태
2. 설정과 언어
3. 티어 / 구독 상태
4. admin이면 추가 운영 섹션

### 블록 구조

1. Identity Zone
2. Settings Zone
3. Tier / Subscription Zone
4. Admin Zone

### 상호작용 톤

- 가장 안정적이고 예측 가능해야 한다
- 과한 연출보다 명확한 라벨과 구조가 중요
- 관리 화면처럼 건조해지지 않도록 미세한 재질감만 유지

### 하지 말아야 할 것

- Mine처럼 hero scene으로 만들기
- Vault처럼 자산 전시 공간처럼 보이게 만들기
- Lab처럼 툴 화면처럼 과도하게 복잡하게 만들기

### 3초 판별 질문

- 계정/설정 허브로 바로 이해되는가?
- 조용하지만 premium한 신뢰감이 있는가?
