# V2 Roadmap

> Gate-Based Development. 검증 → 판단 → 진행. 일직선이 아니라 Gate마다 멈추고 확인한다.

---

## 전체 구조

```
Phase 0 (웹 기반)     → Sprint 0: 디자인 방향 + 시스템 + Auth + 레이아웃
Phase 1 (핵심 루프)    → Sprint 1: Mine 기능 + 공간감 / Sprint 2: Vault-Lab 기능 + 공간감
                       → Gate 1: 루프 완주율 + "다시 쓰고 싶다" 판단
Phase 1.5 (경험 품질)  → Sprint 3: 프리미엄 모션 + Sprint 4: 리텐션/랜딩
                       → Gate 2: D1 재방문율 > 20%
Phase 2 (수익화)       → Sprint 5: 티어/결제 + Sprint 6: Pro 가치
                       → Gate 3: Free→Pro 전환율 > 2%
Phase 3 (확장)         → Gate 3 통과 후 범위 결정
```

## 핵심 원칙

- Gate를 통과하지 못하면 다음이 아니라 **현재를 개선**
- Phase 0~1은 확정. 이후는 Gate 결과에 따라 조정
- 백엔드/DB는 이미 완성. 로드맵의 주 작업은 **웹 프론트엔드**
- 디자인은 보조 트랙이 아니라 각 Sprint의 동등한 완료 조건
